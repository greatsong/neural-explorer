# 5부 · 6부 실습 안내 가이드

> Artificial Neural Net Explorer의 5부(분류를 넘어 생성으로) · 6부(언어를 다루는 신경망)
> 두 개의 히든 스테이지를 학생 손끝에서 따라가는 실습 가이드입니다.
> 1~4부 운영은 [`teaching-guide.md`](./teaching-guide.md)를 보세요.

## 두 개의 히든 스테이지

5부와 6부는 4부·5부를 끝낸 학생만 발견하는 "포털" 너머에 숨어 있습니다.

| 단계 | 진입 조건 | 입구 | 컬러 톤 |
|---|---|---|---|
| **5부 — 분류를 넘어 생성으로** | 4부(11·12) 완료 | `12. MNIST 도전` 페이지 끝 1차 포털 | 보라 + 시안 |
| **6부 — 언어를 다루는 신경망** | 5부(13·14) 완료 | `14. 오토인코더` 페이지 끝 2차 포털 (책이 부서지는 모티프) | 호박 + 분홍 |

## 5부 — 분류를 넘어 생성으로

### 페이즈 13. 평균과 분포 — 가장 단순한 생성 모델

**학습 목표**: 같은 라벨 그림의 픽셀을 평균내면 "평균적인 모양"이 나오고, 그게 가장 단순한 생성 모델임을 체감.

![p13](screenshots/p13-mean.png)

**해보세요**
1. 데이터셋 버튼(네모/원, 동물/탈것 등)을 바꿔가며 두 라벨의 평균 이미지가 어떻게 다른지 비교.
2. **혼합 슬라이더(blend)** 를 0 ↔ 1로 옮기며 두 평균이 어떻게 섞이는지 관찰.
3. **노이즈 슬라이더(noise)** 를 올려 똑같은 평균에서 시드별로 다른 그림이 나오는 걸 확인.

**짚어줄 점**
- 분산 패널에서 색이 진한 픽셀일수록 "그림마다 자주 바뀌는 자리"라는 뜻.
- 분산이 큰 자리에 노이즈를 더 주면 더 그럴듯한 새 그림이 나옵니다.

---

### 페이즈 14. 오토인코더 — 잠재 공간으로 그림 만들기

**학습 목표**: 그림 → 짧은 코드(잠재 벡터) → 그림 으로 자기 자신을 복원하는 신경망 두 개를 학습. 디코더가 곧 "코드 → 그림" 마법 함수가 됨.

![p14 학습](screenshots/p14-train.png)
*② 학습 탭 — 학습 시작을 눌러 인코더·디코더 가중치를 동시에 학습합니다.*

![p14 잠재 공간](screenshots/p14-explore.png)
*③ 잠재 공간 탐험 — 잠재 코드(z₁, z₂) 슬라이더를 옮기면 디코더가 새 그림을 그려 줍니다.*

**해보세요**
1. ① 구조 이해 탭에서 64픽셀 → 12 은닉 → **2차원 잠재 코드** → 12 은닉 → 64픽셀 흐름 확인.
2. ② 학습 탭에서 **학습 시작** 버튼. 손실이 내려가는 걸 확인.
3. ③ 잠재 공간 탐험에서 z₁·z₂ 슬라이더를 직접 만져 새 그림을 만들어 보기.
4. (도전) 학습 데이터에 없던 자리에 z를 두면 어떤 모양이 나오는지 살펴보기.

**짚어줄 점**
- 잠재 공간의 한 점 = 한 그림이라는 직관.
- 이게 Stable Diffusion · VAE · DALL·E의 출발점입니다.

> 💡 **여기서 2차 포털이 열립니다.** 페이지 맨 아래의 "책의 균열"을 통과하면 6부로 진입.

## 6부 — 언어를 다루는 신경망

5부까지는 모델이 "그림"을 다뤘다면, 6부는 모델이 "말과 글"을 다루는 차원입니다.
글자 → 토큰 → 임베딩 → 시퀀스 → 어텐션 → 트랜스포머 → GPT 다음 토큰 까지 한 단계씩 따라갑니다.

### 페이즈 15. 텍스트가 숫자가 되기까지

**학습 목표**: 모든 글자는 결국 숫자(코드포인트)로 바뀌고, UTF-8로 1~4 바이트 묶음이 됨을 체감.

![p15 코드포인트](screenshots/p15-codepoint.png)
*① 글자 → 코드포인트 — 글자 클릭만으로 10진/16진/UTF-8 바이트/2진수가 펼쳐집니다.*

![p15 직접 입력](screenshots/p15-input.png)
*② 직접 입력 — 자신이 친 텍스트의 글자별 표가 즉시 채워집니다.*

![p15 비교](screenshots/p15-compare.png)
*③ 비교 — 영/한/한자/이모지의 글자당 바이트가 1.00 → 3.67 까지 점점 커집니다.*

**해보세요**
1. ② 탭 입력란에 본인 이름을 한글·영어·로마자로 모두 쳐 넣고 표를 비교.
2. 🚀, 🤖 같은 이모지를 넣고 한 글자가 **4 바이트**임을 확인.
3. ③ 탭에서 "GPT는 🤖 입니다" 같은 섞인 문장의 글자당 바이트가 영어·한글·이모지 사이 어디쯤인지 살펴보기.

---

### 페이즈 16. 토큰 — 단어보다 작고, 글자보다 큰 조각

**학습 목표**: 모델은 글자도 단어도 아닌 "토큰" 단위로 입력을 받음. BPE 규칙으로 어떻게 토큰이 만들어지는지 직관 잡기.

![p16 토크나이저](screenshots/p16-tokenizer.png)
*① GPT 스타일 토크나이저 시연 — 같은 문장이 영어·한글에서 다르게 잘립니다.*

![p16 BPE](screenshots/p16-bpe.png)
*② BPE 미니 시뮬레이터 — 단계별 합치기 과정을 한 클릭씩 진행.*

![p16 비교](screenshots/p16-compare.png)
*③ 영어 vs 한글 — 같은 의미라도 한글은 토큰 수가 훨씬 많아집니다.*

**해보세요**
1. ② 탭에서 **다음 합치기** 버튼을 한 번씩 누르며 'low_', 'er_', 'low'가 차례로 한 토큰이 되는 걸 관찰.
2. 한 행의 단어가 어떻게 압축되는지 색칠된 토큰 칩으로 확인.
3. ③ 탭에서 같은 의미의 영어/한글 문장의 **글자당 토큰 수**를 비교 (보통 한글이 2배 이상).

**짚어줄 점**
- 한글이 더 잘게 쪼개지는 건 "GPT 사전이 영어 코퍼스로 학습됐기 때문"이라는 점.
- API 비용이 한글에서 더 비싸지는 실제 이유입니다.

### 페이즈 17. 원-핫 → 임베딩, 단어가 벡터가 되는 이유

**학습 목표**: 원-핫의 한계(고차원 + 모든 쌍 cos = 0) → 학습되는 임베딩 벡터로 의미를 위치에 새기는 발상.

![p17 임베딩 행렬 + 3D](screenshots/p17-embed.png)
*② 임베딩 행렬 W — 6개 단어 × 3차원 행렬과 회전 가능한 3D 산포도.*

![p17 슬라이더](screenshots/p17-play.png)
*③ 직접 만져보기 — 두 단어 벡터를 슬라이더로 옮기면 cos 유사도가 즉시 변합니다.*

**해보세요**
1. ② 탭의 3D 그래프를 **드래그**해서 동물 / 과일 / 컴퓨터 클러스터 위치 확인. (자동 회전 중이면 첫 드래그에 멈춤)
2. ③ 탭에서 "고양이"와 "강아지"를 골라 슬라이더로 한 단어를 멀리 보냈을 때 cos가 1 → -1까지 변하는 걸 관찰.
3. 막대 차트에서 단어 A 기준으로 다른 모든 단어와의 유사도가 함께 변하는 걸 확인.

---

### 페이즈 18. Word2Vec 미니 — 브라우저에서 직접 학습

**학습 목표**: skip-gram 알고리즘으로 임베딩이 "자동으로" 자리잡는 과정을 직접 학습 + 벡터 산수의 의미.

![p18 학습 + 손실](screenshots/p18-train.png)
*① 학습 — 학습 시작을 누르면 손실이 내려가는 그래프를 볼 수 있습니다.*

![p18 PCA 3D](screenshots/p18-space.png)
*② 임베딩 공간 — 학습된 16차원 벡터를 PCA로 3D 투영. 왕족·가족·동물·과일 클러스터가 보입니다.*

![p18 벡터 산수](screenshots/p18-arith.png)
*③ 벡터 산수 — 검증된 페어 한 번 클릭으로 "왕 - 남자 + 여자 ≈ ?" 결과를 볼 수 있습니다.*

**해보세요**
1. 영어 / 한글 토글 + 시드(EN=7, KO=100 권장) + 스텝(1500~2000) 으로 **학습 시작**.
2. ② 탭에서 클러스터 회전. 왕족(보라), 동물(녹색), 과일(주황)이 서로 떨어진 자리에 모이는지 확인.
3. ③ 탭의 검증 페어 버튼 클릭:
   - **boy - man + woman ≈ girl** (top-1)
   - **king - man + woman ≈ queen** (top-2~3)
   - **왕 - 남자 + 여자 ≈ 여왕** (top-1)
4. 자유 입력으로 다른 페어를 시도해 **흔들리는 결과**도 함께 보기.

**짚어줄 점**
- 안정적인 결과를 위해 진짜 Word2Vec은 수십억 단어를 학습한다는 점.
- "데이터가 많을수록 의미의 위치가 또렷해진다."

### 페이즈 19. 시퀀스 — 순서가 의미를 만드는 순간

**학습 목표**: 단어 가방 / 평균 임베딩의 한계 → RNN이 순서를 어떻게 보존하는지 직관.

![p19 평균](screenshots/p19-avg.png)
*② 평균 임베딩의 한계 — "I love you"와 "you love I"의 평균 벡터가 정확히 같습니다.*

![p19 RNN](screenshots/p19-rnn.png)
*③ RNN — carry 슬라이더로 이전 상태를 얼마나 유지할지 직접 조절.*

**해보세요**
1. ① 탭에서 "Dog bites man" vs "Man bites dog" 같은 페어를 보며 순서가 의미를 어떻게 뒤집는지 토론.
2. ③ 탭의 carry 슬라이더를 0 → 0.5 → 0.9 로 변경하며 두 문장의 **최종 기억** 값이 어떻게 갈라지는지 확인.
   - carry=0 : 마지막 입력만 기억 (정/역 순서 차이 큼)
   - carry=0.9 : 첫 입력만 기억 (새 입력 거의 무시)
3. 셀 내부의 막대 그래프로 t=1, t=2, t=3에서 기억이 어떻게 누적되는지 관찰.

### 페이즈 20. 어텐션 — 어디에 집중할지 정해보기

**학습 목표**: 어텐션은 "단어들끼리 점수 매기기 + softmax로 정규화 + 가중 합"이라는 세 단계임을 손으로 만져보며 이해.

![p20 행렬](screenshots/p20-matrix.png)
*② 어텐션 행렬 만지기 — 각 칸의 슬라이더로 점수를 직접 조정.*

![p20 가중 합](screenshots/p20-output.png)
*③ 가중 합 결과 — 어텐션 가중치로 단어 벡터를 합쳐 새로운 문맥 표현을 만듭니다.*

**해보세요**
1. ② 탭에서 "봤다" 행의 점수를 만져 "고양이는" vs "쥐를" 중 누구를 보는지 옮겨보기.
2. softmax 가중치 표에서 **각 행의 합이 1**인 걸 확인 (반올림으로 1.01도 가능).
3. ③ 탭의 다이어그램에서 입력 → 출력 화살표 굵기가 가중치라는 걸 관찰. "봤다'"의 출력이 "고양이는" 색으로 살짝 물든 걸 확인.

**짚어줄 점**
- 실제 식 `softmax(QKᵀ/√d)V`을 Q=K=V로 단순화한 시뮬레이션이지만, 본질은 동일.

### 페이즈 21. 멀티헤드 트랜스포머 — 여러 시선과 한 블록의 흐름

**학습 목표**: 한 어텐션은 한 시선밖에 못 봄 → 여러 헤드 + 한 블록(LayerNorm → Attn → 잔차 → LayerNorm → FFN → 잔차) → 깊이 쌓기.

![p21 4개 헤드](screenshots/p21-heads.png)
*① 4개 헤드의 시선 — "철수가 영희에게 책을 줬다"를 주격/대상/목적어/전체 헤드가 다르게 봅니다.*

![p21 블록](screenshots/p21-block.png)
*② 트랜스포머 블록 — 한 블록의 데이터 흐름과 두 개의 잔차 연결.*

![p21 깊이](screenshots/p21-depth.png)
*③ 1층 vs 6층 — 슬라이더로 층 수를 늘리면 깊은 층이 추상적 관계까지 잡습니다.*

**해보세요**
1. ① 탭에서 4개 헤드 중 **어떤 헤드가 어떤 관계**를 보는지 짝과 맞춰 보기.
2. ② 탭의 블록 다이어그램을 따라가며 잔차 연결이 왜 두 군데에 있는지 토론.
3. ③ 탭의 층 슬라이더를 1 → 12로 옮기며 GPT-2(12층), GPT-3(96층) 정도의 규모를 가늠.

### 페이즈 22. GPT의 다음 토큰 — 샘플링이 곧 창의성

**학습 목표**: GPT가 한 토큰씩 만드는 방식 — logit → softmax 분포 → temperature/top-k/top-p로 샘플링.

![p22 어휘 분포](screenshots/p22-logits.png)
*① 어휘 분포 — 짧은 문맥 다음에 올 토큰의 top-10 확률.*

![p22 샘플링](screenshots/p22-sampling.png)
*② 샘플링 슬라이더 — temperature/top-k/top-p가 분포를 어떻게 깎고 평탄화하는지.*

![p22 자기회귀](screenshots/p22-generate.png)
*③ 자기회귀 생성 — 한 토큰씩 누적해 시퀀스를 늘려 갑니다.*

**해보세요**
1. ② 탭에서 **temperature** 를 0.1 → 1 → 2 로 옮기며 top-1 확률이 99.8% → 38% → 17% 로 변하는 걸 확인.
2. **top-k** 를 5 정도로 줄여서 후보가 줄어드는 걸 관찰.
3. ③ 탭에서 프롬프트("고양이는", "오늘 날씨가" 등)를 골라 **한 토큰 더** 버튼을 여러 번. 같은 시드면 결과가 동일, 새 시드를 누르면 달라지는 걸 확인.
4. T를 매우 낮게/높게 두고 결과의 차이를 직접 비교.

**마무리 멘트(권장)**
> "지금까지 본 모든 단계 — 토큰화, 임베딩, 어텐션, 멀티헤드, 깊은 블록, 다음 토큰 분포 — 가
> ChatGPT가 한 줄을 답할 때 한 토큰마다 한 번씩 통째로 돌아갑니다."
