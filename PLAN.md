# neural-explorer 1~4부 재구성 계획

> 5·6부(생성·언어, 히든 스테이지)는 본 계획의 범위 밖이며 그대로 유지한다.
> 본 문서는 **코덱스 검토 결과를 반영한 작업 계획**이며, 선결 작업 트리 정리 후 실작업에 들어간다.

---

## 0. 목표 (Why)

현재 1~4부는 *개념 단위* 페이즈(P1 해부, P2 순전파, P3 손실+경사하강법, P4 학습률, P5 역전파…)로 구성되어 있다. 그러나 학생 입장에서 알고리즘이 한 덩어리로 잡히지 않는다. 특히 **P5 한 페이즈에 5탭(직관·수식·유도·종합·실생활)이 압축**되어 너무 많은 일이 일어나고 있다.

새 구조는 *작동 순서 단위*로 흐름을 재배치한다.

- **A. 알고리즘의 이해** — 예측 → 오차 → 내려가는 방향 → 기울기 → 전체 루프 → 실생활 적용
- **B. 데이터 수집·학습·평가** — 머신러닝의 기본 작업 흐름 (현재 3부 + 2부 평가 도구)
- **C. 모델 개선·일반화** — 결과를 더 낫게 만드는 반복 (현재 4부 + 2부 개선 도구)

## 3. 핵심 결정 사항 — 코덱스 검토 결과

### 결정 1. **A5에 직관·종합을 함께 둘지, 직관만 둘지**

현 P5 탭1(직관)과 탭4(종합)은 둘 다 *왼쪽 다이어그램 + 오른쪽 손실 곡선*의 통합 뷰이지만, 탭1은 학습 컨트롤만 있고 탭4는 + 식 카드까지 함께 보여준다. A5를 하나로 묶으면 식 카드를 포함한 탭4 쪽으로 가는 게 자연스러우나, 그러면 탭1의 *식이 등장하기 전 직관* 단계가 사라진다.

> **선택지** (a) A5는 탭1+탭4 모두 포함, 같은 페이즈 안에서 단계 전환 / (b) 탭4만 A5로, 탭1은 A2~A3 단계 사이의 *직관 박스*로 흩뿌림 / (c) A4 앞에 A4½ 직관 페이즈 추가(=총 17페이즈)

**결정: (a).** A5는 탭1과 탭4를 모두 품되, 별도 페이즈를 추가하지 않는다. A5 내부를 `1. 직관 리플레이` → `2. 종합 식 카드`의 짧은 단계 전환으로 만든다. A4에서 수식과 평균 계산을 이미 다뤘기 때문에, A5는 "식이 처음 등장하는 곳"이 아니라 "알고리즘 한 바퀴가 실제로 돌아가는 곳"으로 역할을 고정한다.

### 결정 2. **P10 분할의 깊이**

현 P10은 그림 그리기 → 학습 버튼 → 갤러리 공유가 한 페이지에 흐른다. B2/B3/B4로 쪼개면 학생이 같은 캔버스에 세 번 들어와야 한다. 같은 인터랙션을 페이즈 사이에서 *상태로 연속*시킬지, 페이즈마다 새로 시작할지 결정해야 한다.

> **선택지** (a) 한 페이즈에서 단계 탭으로 / (b) 페이즈를 쪼개되 캔버스 상태 공유 / (c) 분할 포기, B2~B4를 한 페이즈로

**결정: (b).** B2/B3/B4는 페이즈를 나누되 도트 데이터 상태를 공유한다. 캔버스는 B2에 집중시키고, B3는 수집된 데이터의 요약·정제·학습/평가 분할, B4는 같은 데이터로 학습·테스트·공유를 담당한다. 데이터가 부족하면 프리셋 불러오기와 B2로 돌아가기 링크를 제공한다.

### 결정 3. **P4 학습률을 어디에 넣을지**

코덱스 의견: A3 또는 A5. 내 추가 의견: **A3이 정답.** A5에 넣으면 이미 너무 많은 게 돌아간다. A3 안에 *η 슬라이더 실험 박스*로 압축하면 학습률의 직관을 잃지 않으면서 페이즈는 줄일 수 있다.

> **확인 필요**: A3에 슬라이더를 넣어도 현 P4의 *발산 vs 수렴 시각화*가 살아남는가?

**결정: A3.** 현 P4의 핵심인 `η = 1.2 / 1.0 / 0.1 / 0.01` 비교 시각화는 살린다. 다만 독립 페이즈가 아니라 A3 안의 `보폭 실험` 섹션 또는 내부 탭으로 압축한다. A3의 흐름은 `기울기 반대 방향` → `보폭 η` → `발산/진동/수렴/느림 비교`가 된다. A5에서는 η를 새로 가르치지 않고 전체 식 안에서만 다시 만난다.

### 결정 4. **P6 입시 합격 예측의 위상**

현재 2부의 첫 페이즈로 "분류 문제의 첫 만남" 역할. 새 구조에서는 B1(문제 정의) 안의 분류 예시로 들어간다. 그러면 회귀(A6 기온)와 분류(B1 입시)가 깔끔하게 대비되는 장점이 있다. 다만 P6는 *결정 경계 시각화*가 핵심 인터랙션이라, B1 안에 넣을지 별도 페이즈로 살릴지 판단 필요.

> **선택지** (a) B1에 흡수, 결정 경계는 C1로 미루기 / (b) B1에 흡수하되 결정 경계는 그 자리에서 보여줌 / (c) B0 별도 페이즈

**결정: (b).** B1에 흡수하되 결정 경계와 합격컷 인터랙션은 남긴다. B1은 단순 소개 페이지가 아니라 `회귀에서 분류로 넘어가는 첫 모델`이다. 단, P6의 모든 평가 설명을 다 끌고 오지는 않는다. B1에서는 라벨·특징·가중치·합격컷·결정 경계까지만 잡고, 정확도/혼동 행렬은 B5, 정밀도/재현율은 C3으로 보낸다.

### 결정 5. **혼동 행렬의 위치**

새 B5 = "정확도와 혼동 행렬". 현재 P8에는 이미 기본 혼동 행렬이 들어 있으므로 완전 신설이 아니라 재정리와 일반화가 핵심이다. 내용량이 작지 않다.

> **확인 필요**: B5를 한 페이즈로 묶을지(=정확도+혼동행렬), 두 페이즈로 쪼갤지.

**결정: 한 페이즈.** 정확도와 혼동 행렬은 한 화면에서 같이 봐야 "맞춘 개수"와 "어떻게 틀렸는지"가 연결된다. 대신 정밀도·재현율·임계값은 C3으로 넘겨 B5의 밀도를 낮춘다. B5가 길어지면 내부 progressive disclosure로 처리하고, 페이즈를 더 쪼개지는 않는다.

### 결정 6. **B/C의 주 데이터 흐름을 어떻게 통일할지**

검토 중 가장 큰 리스크는 B가 도트 데이터(P10)와 입시 데이터(P6/P8)를 함께 끌어온다는 점이다. 학생에게 "지금 어떤 모델을 평가하는가"가 흐려질 수 있다.

**결정: B의 본선은 도트 데이터로 통일한다.** B2에서 모은 도트 데이터가 B3의 분할, B4의 학습, B5의 평가로 이어진다. 입시 예측은 B1에서 분류·결정 경계를 여는 대표 케이스로 쓰고, C1에서 "데이터를 더 모으면 일반화가 어떻게 달라지는가"를 보여주는 케이스로 다시 사용한다. 즉, B1/C1은 입시 케이스, B2~B5는 도트 케이스로 명확히 라벨링한다.

## 5. 작업 순서 (제안)

1. **선결 — 작업 트리 정리**: 현재 40+ 변경 파일이 미커밋. 본 작업 들어가기 전에 기존 변경분을 별도 커밋으로 마무리하거나 stash 필요. (덮어쓰기 위험)
2. **타입/메타 레이어**: phases.ts → toc.ts → guideContent.ts 순으로 새 ID 체계 도입. visible ID는 `a1~a6`, `b1~b5`, `c1~c5`를 쓰고 히든 스테이지는 기존 `p13~p22`를 유지. 이 단계에서 앱이 빌드는 되되 일부 페이즈는 Stub.
3. **진행도 마이그레이션**: 기존 `p1~p12` 완료 상태를 새 ID로 옮길지, 아니면 재구성 안내와 함께 리셋할지 결정 후 구현. 운영 중인 수업 데이터가 있으면 마이그레이션 우선.
4. **A 영역**: P1+P2 → A1 / P3 분할 → A2+A3 / P5 분할 → A4+A5+A6 / P4 흡수
5. **B 영역**: B1에서 P6 핵심 흡수 / P10 분할과 공유 상태 도입 / B5에서 B4 모델 평가
6. **C 영역**: P7 → C1 / C2 신설 / P9 → C3 / P11 → C4 / P12 → C5
7. **교과서 본문**: 새 slug에 맞춰 P*.tsx 재배치, 본문 일부 다듬기
8. **print HTML/PDF**: A/B/C + overview + problems 변형을 마지막에 일괄 재생성
9. **회귀 테스트**: 히든 스테이지 게이트, 모바일 드로어, PDF 경로, 도트 상태 공유

## 7. 코덱스 검토 답변 (추가 포인트)

1. **A5 직관·종합**: 통합이 자연스럽다. 다만 한 화면에 처음부터 식을 다 보이지 말고 `직관 리플레이` 후 `종합 식 카드`로 단계 노출한다.
2. **P10 분할 깊이**: 페이즈 분할은 괜찮다. 부담은 "캔버스를 반복해서 그리게 하는 것"에서 오므로, 캔버스는 B2에만 두고 B3/B4는 같은 상태를 이어받는다.
3. **P6 위상**: B1에 흡수하되 핵심 인터랙션은 살린다. B1이 회귀(A6)에서 분류(B)로 넘어가는 다리 역할을 한다.
4. **혼동 행렬**: B5 한 페이즈로 충분하다. 정확도와 혼동 행렬은 함께 봐야 하고, 정밀도·재현율은 C3으로 넘긴다.
5. **누락 가능 개념**: 활성화 함수와 편향은 A1에 반드시 남긴다. 학습률은 A3, 기울기 평균은 A4, 전체 갱신은 A5, 특징 스케일링/연도 offset은 A6의 짧은 주석으로 남긴다. 분류 손실(BCE/시그모이드)은 B4 또는 C3의 선택 심화로만 다룬다.
6. **회귀 → 분류 전환**: B1이 맡는다. A6 끝에서 "숫자를 예측하는 회귀"를 마치고, B1 첫 문단에서 "이제 숫자가 아니라 라벨을 맞힌다"로 전환하면 충분하다.
7. **페이즈 수 증가**: visible 16개는 감당 가능하다. 모바일 드로어는 A/B/C 그룹 접기 또는 짧은 그룹 헤더를 쓰고, 히든 10개는 지금처럼 메뉴에서 숨긴다.
8. **출판 단위**: 새 visible 교재는 A/B/C 세 권이 가장 가볍다. `overview-assessment`와 `student-problems`는 별도 묶음으로 재생성하고, 5·6부 PDF는 공유 컴포넌트 변화가 없으면 그대로 둔다.
