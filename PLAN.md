# neural-explorer 1~4부 재구성 계획

> 5·6부(생성·언어, 히든 스테이지)는 본 계획의 범위 밖이며 그대로 유지한다.
> 본 문서는 **코덱스 검토 결과를 반영한 작업 계획**이며, 선결 작업 트리 정리 후 실작업에 들어간다.

---

## 0. 목표 (Why)

현재 1~4부는 *개념 단위* 페이즈(P1 해부, P2 순전파, P3 손실+경사하강법, P4 학습률, P5 역전파…)로 구성되어 있다. 그러나 학생 입장에서 알고리즘이 한 덩어리로 잡히지 않는다. 특히 **P5 한 페이즈에 5탭(직관·수식·유도·종합·실생활)이 압축**되어 너무 많은 일이 일어나고 있다.

새 구조는 *작동 순서 단위*로 흐름을 재배치한다.

- **A. 알고리즘의 이해** — 예측 → 오차 → 내려가는 방향 → 기울기 → 전체 루프 → 실생활 적용
- **B. 데이터 수집·학습·분류 출력** — 기본 데이터셋 정제 → 이진 분류 → 다중 분류와 소프트맥스
- **C. 딥러닝** — 심층 신경망이 문제를 어떻게 해결하는지 — 역전파와 MNIST

## 3. 핵심 결정 사항 — 코덱스 검토 결과

### 결정 1. **A5에 직관·종합을 함께 둘지, 직관만 둘지**

현 P5 탭1(직관)과 탭4(종합)은 둘 다 *왼쪽 다이어그램 + 오른쪽 손실 곡선*의 통합 뷰이지만, 탭1은 학습 컨트롤만 있고 탭4는 + 식 카드까지 함께 보여준다. A5를 하나로 묶으면 식 카드를 포함한 탭4 쪽으로 가는 게 자연스러우나, 그러면 탭1의 *식이 등장하기 전 직관* 단계가 사라진다.

> **선택지** (a) A5는 탭1+탭4 모두 포함, 같은 페이즈 안에서 단계 전환 / (b) 탭4만 A5로, 탭1은 A2~A3 단계 사이의 *직관 박스*로 흩뿌림 / (c) A4 앞에 A4½ 직관 페이즈 추가(=총 17페이즈)

**결정: (a).** A5는 탭1과 탭4를 모두 품되, 별도 페이즈를 추가하지 않는다. A5 내부를 `1. 직관 리플레이` → `2. 종합 식 카드`의 짧은 단계 전환으로 만든다. A4에서 수식과 평균 계산을 이미 다뤘기 때문에, A5는 "식이 처음 등장하는 곳"이 아니라 "알고리즘 한 바퀴가 실제로 돌아가는 곳"으로 역할을 고정한다.

### 결정 2. **P10 분할의 깊이**

현 P10은 그림 그리기 → 학습 버튼 → 갤러리 공유가 한 페이지에 흐른다. B2/B3/B4로 쪼개면 학생이 같은 캔버스에 세 번 들어와야 한다. 같은 인터랙션을 페이즈 사이에서 *상태로 연속*시킬지, 페이즈마다 새로 시작할지 결정해야 한다.

> **선택지** (a) 한 페이즈에서 단계 탭으로 / (b) 페이즈를 쪼개되 캔버스 상태 공유 / (c) 분할 포기, B2~B4를 한 페이즈로

**결정: (b).** B2/B3/B4는 페이즈를 나누되 도트 데이터 상태를 공유한다. B2에는 기본 데이터셋을 제공하되 일부 샘플은 라벨 오류·모호한 그림·노이즈처럼 전처리 판단이 필요한 상태로 둔다. 캔버스는 B2에 집중시키고, B3는 수집/정제된 데이터의 요약과 학습/평가 분할, B4는 같은 데이터로 이진 분류 모델을 학습하는 역할을 맡는다. 갤러리 공유는 제거한다.

### 결정 3. **P4 학습률을 어디에 넣을지**

코덱스 의견: A3 또는 A5. 내 추가 의견: **A3이 정답.** A5에 넣으면 이미 너무 많은 게 돌아간다. A3 안에 *η 슬라이더 실험 박스*로 압축하면 학습률의 직관을 잃지 않으면서 페이즈는 줄일 수 있다.

> **확인 필요**: A3에 슬라이더를 넣어도 현 P4의 *발산 vs 수렴 시각화*가 살아남는가?

**결정: A3.** 현 P4의 핵심인 `η = 1.2 / 1.0 / 0.1 / 0.01` 비교 시각화는 살린다. 다만 독립 페이즈가 아니라 A3 안의 `보폭 실험` 섹션 또는 내부 탭으로 압축한다. A3의 흐름은 `기울기 반대 방향` → `보폭 η` → `발산/진동/수렴/느림 비교`가 된다. A5에서는 η를 새로 가르치지 않고 전체 식 안에서만 다시 만난다.

### 결정 4. **P6 입시 합격 예측의 위상**

현재 2부의 첫 페이즈로 "분류 문제의 첫 만남" 역할을 했지만, 새 구조에서는 도트 데이터 하나로 B/C 본선을 통일한다. 입시 예측은 좋은 실생활 예시지만, 딥러닝 기초의 필수 축인 입력 벡터·출력층·소프트맥스·모델 복잡도와 직접 이어지지 않아 본선에서 제외한다.

> **선택지** (a) B1에 흡수 / (b) 선택 심화로 이동 / (c) 완전 제외

**결정: (c).** 입시 예측은 완전히 제외한다. B1은 동그라미/세모 도트 분류 문제로 바로 시작한다. 회귀(A6)에서 분류(B)로 넘어가는 전환은 "숫자를 맞히는 문제에서 라벨을 맞히는 문제로 바뀐다"는 짧은 설명으로 충분하다.

### 결정 5. **혼동 행렬과 정밀도·재현율의 위치**

기존 계획은 B5에서 정확도와 혼동 행렬을 다루고, C3에서 정밀도·재현율·임계값을 다루는 구성이었다. 그러나 딥러닝 기초 개념만 놓고 보면 이 둘은 평가 리터러시에 가깝고, 출력층·다중 분류·소프트맥스보다 우선순위가 낮다.

> **선택지** (a) B5에 혼동 행렬 유지, C3만 제거 / (b) 혼동 행렬과 C3 모두 핵심에서 제거 / (c) 둘 다 선택 심화로 이동

**결정: (b)에 가깝게 핵심에서 제거.** 핵심 페이즈에서는 정확도와 오답 예시만 남기고, 혼동 행렬·정밀도·재현율·임계값은 다루지 않는다. 필요하면 교사용 가이드나 선택 심화 박스에 "어떤 클래스를 어떤 클래스로 헷갈렸는지 표로 볼 수도 있다" 정도만 남긴다.

### 결정 6. **B/C의 주 데이터 흐름을 어떻게 통일할지**

검토 중 가장 큰 리스크는 B가 도트 데이터(P10)와 입시 데이터(P6/P8)를 함께 끌어온다는 점이었다. 학생에게 "지금 어떤 모델을 평가하는가"가 흐려질 수 있기 때문이다.

**결정: B/C의 본선은 도트 데이터로 통일한다.** B1에서 문제와 라벨을 정의하고, B2에서 기본 데이터셋을 정제하고, B3에서 학습/평가 데이터를 나누고, B4에서 2종 분류를 학습하고, B5에서 3종 분류와 소프트맥스로 확장한다. C1의 일반화 설명도 같은 도트 데이터의 평가 결과 위에서 처리한다.

### 결정 7. **동그라미·세모에서 동그라미·세모·네모로 확장할지**

딥러닝 기초 관점에서는 이 확장이 매우 좋다. 단일 뉴런/이진 분류에서 멈추면 "신경망의 출력층이 문제에 맞게 달라진다"는 핵심이 잘 보이지 않는다.

**결정: B4는 2종 분류, B5는 3종 분류로 확장한다.** B4에서 동그라미 vs 세모를 학습하고, B5에서 네모를 추가해 출력 뉴런이 2개에서 3개로 늘어나는 모습을 보여준다. 각 출력 뉴런은 "그 클래스라고 보는 점수(logit)"를 내고, 소프트맥스는 이 점수들을 합이 1인 확률처럼 바꿔 주는 함수로 짧게 소개한다. 수식은 선택적으로만 노출한다.

- B4 2출력 모델, B5 3출력 모델의 출력 뉴런 수와 소프트맥스 표시가 일관되는지

## 5. 작업 순서 (제안)

1. **선결 — 작업 트리 정리**: 현재 40+ 변경 파일이 미커밋. 본 작업 들어가기 전에 기존 변경분을 별도 커밋으로 마무리하거나 stash 필요. (덮어쓰기 위험)
2. **타입/메타 레이어**: phases.ts → toc.ts → guideContent.ts 순으로 새 ID 체계 도입. visible ID는 `a1~a6`, `b1~b5`, `c1~c4`를 쓰고 히든 스테이지는 기존 `p13~p22`를 유지. 이 단계에서 앱이 빌드는 되되 일부 페이즈는 Stub.
3. **진행도 마이그레이션**: 기존 `p1~p12` 완료 상태를 새 ID로 옮길지, 아니면 재구성 안내와 함께 리셋할지 결정 후 구현. 운영 중인 수업 데이터가 있으면 마이그레이션 우선.
4. **A 영역**: P1+P2 → A1 / P3 분할 → A2+A3 / P5 분할 → A4+A5+A6 / P4 흡수
5. **B 영역**: B1 도트 문제 정의 / P10 분할과 공유 상태 도입 / B4 2종 분류 / B5 3종 분류와 소프트맥스
6. **C 영역**: P8 일부 → C1 / C2 신설 / P11 → C3 / P12 → C4. P6/P7/P9는 핵심에서 제외
7. **교과서 본문**: 새 slug에 맞춰 P*.tsx 재배치, 본문 일부 다듬기
8. **print HTML/PDF**: A/B/C + overview + problems 변형을 마지막에 일괄 재생성
9. **회귀 테스트**: 히든 스테이지 게이트, 모바일 드로어, PDF 경로, 도트 상태 공유, 2출력/3출력 전환

## 7. 코덱스 검토 답변 (추가 포인트)

1. **A5 직관·종합**: 통합이 자연스럽다. 다만 한 화면에 처음부터 식을 다 보이지 말고 `직관 리플레이` 후 `종합 식 카드`로 단계 노출한다.
2. **P10 분할 깊이**: 페이즈 분할은 괜찮다. 부담은 "캔버스를 반복해서 그리게 하는 것"에서 오므로, 기본 데이터셋을 제공하고 B2에서 전처리 판단을 시킨 뒤 B3/B4/B5가 같은 상태를 이어받는다. 갤러리 공유는 제거한다.
3. **P6 위상**: 입시 예측은 완전히 제외한다. B1은 도트 분류 문제 정의로 바로 시작한다.
4. **혼동 행렬/C3**: 딥러닝 기초 핵심에서는 뺀다. 정확도와 대표 오답 예시만 남기고, 혼동 행렬·정밀도·재현율·임계값은 선택 심화로 분리한다.
5. **누락 가능 개념**: 활성화 함수와 편향은 A1에 반드시 남긴다. 학습률은 A3, 기울기 평균은 A4, 전체 갱신은 A5, 특징 스케일링/연도 offset은 A6의 짧은 주석으로 남긴다. 분류 출력층, 출력 뉴런 수, 소프트맥스는 B5에서 간단히 다룬다.
6. **회귀 → 분류 전환**: B1이 맡는다. A6 끝에서 "숫자를 예측하는 회귀"를 마치고, B1 첫 문단에서 "이제 숫자가 아니라 그림의 라벨을 맞힌다"로 전환하면 충분하다.
7. **페이즈 수 증가**: visible 15개는 감당 가능하다. 모바일 드로어는 A/B/C 그룹 접기 또는 짧은 그룹 헤더를 쓰고, 히든 10개는 지금처럼 메뉴에서 숨긴다.
8. **출판 단위**: 새 visible 교재는 A/B/C 세 권이 가장 가볍다. `overview-assessment`와 `student-problems`는 별도 묶음으로 재생성하고, 5·6부 PDF는 공유 컴포넌트 변화가 없으면 그대로 둔다.

## 9. 구체 계획과 해설

이번 재구성의 핵심 의도는 "좋은 머신러닝 사례를 많이 보여주기"가 아니라, 학생이 **딥러닝의 기본 작동 원리**를 한 줄로 잡도록 만드는 것이다. 따라서 입시 예측, 갤러리 공유, 혼동 행렬, 정밀도·재현율처럼 유익하지만 옆길이 될 수 있는 요소는 본선에서 제외하거나 심화로 내린다. 본선은 다음 한 문장으로 요약된다.

> 뉴런은 예측하고, 오차를 계산하고, 기울기로 가중치를 고치며, 데이터로 일반화되고, 출력층과 은닉층을 늘리며 더 복잡한 문제를 푼다.

### A. 알고리즘의 이해 — "학습 한 step"을 손에 잡히게 만들기

A 영역은 기존 P1~P5의 좋은 시각화와 수식 자산을 재조립한다. 목표는 학생이 `예측 → 오차 → 기울기 → 갱신`을 분리된 개념이 아니라 하나의 반복 루프로 이해하게 하는 것이다.

1. **A1 인공 뉴런의 예측**
   - 현 P1과 P2를 통합한다.
   - 가중치·편향·활성화 함수는 반드시 남긴다.
   - 순전파 퀴즈는 "뉴런이 실제로 ŷ을 만든다"는 계산 감각을 위해 유지한다.
   - 해설 의도: 딥러닝은 거대한 마법이 아니라 작은 뉴런 계산의 반복이라는 출발점을 만든다.

2. **A2 오차와 MSE**
   - 현 P3의 앞부분을 분리한다.
   - 예측값 `ŷ`과 정답 `y`의 차이, 오차 제곱, 평균 제곱 오차를 다룬다.
   - 해설 의도: 학습은 "틀렸다는 느낌"이 아니라 "얼마나 틀렸는지 숫자로 만드는 일"에서 시작한다.

3. **A3 경사하강법**
   - 현 P3의 경사하강법 부분과 P4 학습률을 합친다.
   - 손실이 줄어드는 방향, 기울기의 부호, 학습률 η에 따른 발산/진동/수렴/느림 비교를 한 흐름으로 보여준다.
   - 해설 의도: 학습률은 별도 잡학이 아니라 경사하강법의 보폭이다. 그래서 A5가 아니라 A3에서 가르친다.

4. **A4 기울기 계산하기**
   - 현 P5의 수식 이해와 수식 유도 탭을 승격한다.
   - `dw = 평균(e · x)`, `db = 평균(e)`를 표와 짧은 유도로 보여준다.
   - 해설 의도: "컴퓨터가 알아서 한다"로 넘기지 않고, 가중치가 왜 입력 `x`만큼 책임을 더 지는지 납득하게 한다.

5. **A5 전체 흐름 완성**
   - 현 P5의 직관 탭과 종합 탭을 합친다.
   - 내부 흐름은 `직관 리플레이 → 종합 식 카드`로 둔다.
   - 해설 의도: 식을 처음 배우는 곳이 아니라, 앞에서 배운 예측·오차·기울기·학습률이 한 step으로 돌아가는 순간을 확인하는 곳이다.

6. **A6 기온 예측 프로젝트**
   - 현 P5의 서울 기온 실생활 탭을 독립 프로젝트로 만든다.
   - 회귀 문제로 마무리한다. 연도 offset/스케일링은 짧게 설명한다.
   - 해설 의도: A에서 배운 알고리즘이 장난감 식이 아니라 실제 숫자 예측에 바로 쓰인다는 경험을 준다.

### B. 데이터 수집·학습·분류 출력 — 도트 데이터 하나로 일관되게 가기

B 영역은 입시 예측을 완전히 제외하고, 도트 그림 데이터 하나로 본선을 통일한다. 이유는 분명하다. 딥러닝 기초에서는 "분류 문제의 사회적 맥락"보다 **입력 벡터, 라벨, 출력 뉴런, 소프트맥스**가 더 중요하다.

1. **B1 문제 정의와 라벨**
   - 동그라미 vs 세모 도트 분류로 바로 시작한다.
   - 입력은 8×8 픽셀 64개 숫자, 정답은 라벨이라는 점을 명확히 한다.
   - 해설 의도: A6의 회귀에서 B의 분류로 넘어오며, "숫자를 맞히는 문제"에서 "이름표를 맞히는 문제"로 질문이 바뀌었음을 보여준다.

2. **B2 데이터셋과 전처리**
   - 기본 데이터셋을 제공한다.
   - 일부 샘플에는 라벨 오류, 애매한 그림, 노이즈를 둔다.
   - 학생은 새로 많이 그리기보다, 데이터 품질을 보고 정제한다.
   - 해설 의도: 딥러닝은 모델만의 문제가 아니다. 입력 데이터가 64개 숫자로 바뀌고, 잘못된 라벨은 학습을 망친다는 감각을 만든다.

3. **B3 학습/평가 데이터 나누기**
   - B2에서 정제한 같은 데이터 상태를 이어받는다.
   - 학습 데이터와 평가 데이터를 나누는 이유를 "외운 것인지, 새 그림도 맞히는지 확인"으로 설명한다.
   - 해설 의도: 일반화의 씨앗을 여기서 심는다. 평가는 별도 통계 수업이 아니라, 학습이 진짜인지 확인하는 장치다.

4. **B4 이진 분류 모델 학습**
   - 동그라미 vs 세모를 학습한다.
   - 출력 뉴런 2개를 보여준다. 하나는 동그라미 점수, 하나는 세모 점수다.
   - 갤러리 공유는 제거한다.
   - 해설 의도: 기존 P10의 재미 요소는 덜고, "출력층이 라벨 후보마다 점수를 낸다"는 딥러닝 구조를 선명하게 만든다.

5. **B5 다중 분류와 소프트맥스**
   - 네모를 추가해 동그라미·세모·네모 3종 분류로 확장한다.
   - 출력 뉴런이 2개에서 3개로 늘어나는 모습을 직접 보여준다.
   - 소프트맥스는 "점수들을 합이 1인 확률처럼 바꾸는 출력층 함수"로 짧게 소개한다.
   - 해설 의도: 여기서 학생은 클래스 수가 출력 뉴런 수를 결정한다는 핵심을 배운다. ReLU와 softmax의 역할 차이도 이 지점에서 자연스럽게 잡는다.

### C. 모델 개선·일반화 — 더 깊은 신경망으로 넘어가기

C 영역은 평가 지표를 깊게 파는 대신, 딥러닝의 다음 질문으로 간다. "새 데이터에서도 맞는가?", "너무 외운 것은 아닌가?", "뉴런을 더 쌓으면 무엇이 달라지는가?"

1. **C1 평가와 일반화**
   - B3에서 나눈 평가 데이터로 B4/B5 모델을 확인한다.
   - 정확도와 대표 오답 예시만 사용한다.
   - 혼동 행렬은 본선에서 제외한다.
   - 해설 의도: 평가는 복잡한 지표가 아니라, 새 데이터에서도 통하는지 확인하는 첫 관문이다.

2. **C2 일반화 — 새 데이터에서 틀리는 이유**
   - 학습 데이터에는 잘 맞지만 평가 데이터에서 틀리는 상황을 보여준다.
   - 과적합을 직관적으로 다룬다.
   - 해설 의도: 학생이 "성능이 높다"를 무조건 믿지 않고, 학습 데이터와 새 데이터의 차이를 보게 한다.

3. **C3 모델 복잡도 바꾸기**
   - 현 P11을 재구성한다.
   - 은닉층, 뉴런 수, 모델 용량을 다룬다.
   - 해설 의도: 단일 뉴런과 출력층만으로는 복잡한 패턴을 충분히 못 잡는다. 여기서 "깊은" 신경망으로 넘어가는 이유가 생긴다.

4. **C4 MNIST 도전**
   - 현 P12를 종합 예시로 유지한다.
   - 28×28 입력, 10개 출력 뉴런, 숫자별 확률/점수를 보여준다.
   - 해설 의도: A에서 배운 학습 루프, B에서 배운 출력층/소프트맥스, C에서 배운 모델 복잡도가 실제 손글씨 숫자 분류로 모인다.

### 제외/심화로 내리는 항목

- **입시 합격 예측(P6)**: 완전 제외. 실생활 맥락은 좋지만, 도트 이미지 → 출력층 → 소프트맥스 → MNIST 흐름을 흐린다.
- **데이터 추가 후 재학습(P7)**: 입시 기반이라 본선 제외. 필요하면 일반화 심화 예시로만 보류한다.
- **갤러리 공유**: 제외. 참여 기능이지만 딥러닝 기초 개념과 직접 연결되지 않는다.
- **혼동 행렬, 정밀도·재현율, 임계값(P8/P9 일부)**: 본선 제외. 정확도와 대표 오답만 남기고, 상세 평가 지표는 선택 심화/교사용 가이드로 이동한다.

### 구현 원칙

1. 새 파일을 처음부터 쓰기보다, 기존 P1~P12의 좋은 시각화와 로직을 **부품처럼 이식**한다.
2. visible ID는 `a1~a6`, `b1~b5`, `c1~c4`로 재정의한다.
3. 히든 스테이지 `p13~p22`는 유지한다.
4. B2~C1은 도트 데이터 상태를 공유한다.
5. C3+C4 완료 기준으로 5부 포털 게이트를 갱신한다.
6. print/PDF는 마지막에 A/B/C 단위로 일괄 재생성한다.

## 10. 보강 마이크로카피·시각화 (추후 반영)

> 본 절은 코덱스 + 클로드 비평 종합 → 설계·검수 에이전트 2단 통과 결과물이다. A/B/C 페이즈 실제 작성 시점에 *그대로 박아 넣을 수 있는 수준*으로 다듬어 둔다. 페이즈 추가 없음, 모두 박스 또는 한 줄.
>
> **공통 원칙**: 의인화 금지("AI가 스스로 배운다" X), 제외 개념 밀반입 금지(SGD/배치/정규화/드롭아웃/혼동행렬/정밀도/재현율), 페이즈 참조는 새 ID(A2/A4/A5/B5/C3) 사용.

### 10-1. 마이크로카피 7건

#### 1) B1 — 회귀 → 분류 전환 + cross-entropy 이름 (본문 + 비교 표)

> 지금까지는 "기온 몇 도?"처럼 **숫자 하나**를 맞추는 문제(회귀)를 풀었어요. 이제부터는 "동그라미인가, 세모인가?"처럼 **정해진 종류 중 하나를 고르는 문제**(분류)를 풉니다. 둘은 답 모양이 달라서 오차를 재는 방법도 달라져요 — 회귀는 **A2에서 본 MSE**, 분류는 **교차 엔트로피(cross-entropy)** 라는 손실을 쓰는데, 자세한 내용은 **B5**에서 만나요.

| | 답 | 예 | 손실 |
|---|---|---|---|
| 회귀 | 숫자 | 기온 23.4도 | MSE |
| 분류 | 종류 | 동그라미 | 교차 엔트로피 |

#### 2) B2 — 전처리 전/후 결과 차이 (인터랙션 박스)

> 같은 그림 데이터를 **전처리 없이** 학습시켰을 때와 **전처리 후** 학습시켰을 때를 나란히 비교해 봐요. 정확도 곡선이 어떻게 달라지는지 보세요 — 그림을 깨끗하게 다듬는 일이 모델보다 먼저인 이유가 여기 있습니다.

- 시각화: 좌(원본) / 우(전처리 후) 손실·정확도 곡선 + 최종 정확도 두 숫자
- **고정 시드**(결정론). 캡션 "같은 모델, 같은 학습 시간 (시드 고정)."
- 컴포넌트 재사용: `LossChart`, `ProgressChart`

#### 3) B3 — 평가 데이터 위생 (`aside-warn` 한 줄)

> ⚠ 평가 데이터를 보고 모델 구조를 고치면 **부정행위**예요 — 시험 문제로 공부한 셈이라, 새 데이터에서는 다시 약해집니다.

#### 4) B5 — softmax 경쟁 구조 + cross-entropy 도입

**슬라이더 위 라벨**:
> `z = 출력 뉴런이 가중합으로 만든 점수 (아직 확률 아님)`

**상단 박스 캡션**:
> 한 클래스 점수를 *크게* 올릴수록 다른 두 클래스의 확률이 빠르게 작아져요 — 큰 점수가 더 큰 비율로 자리를 차지합니다(1등 독식). 세 막대 길이의 합은 항상 100%. 이렇게 점수를 "서로 경쟁하는 확률"로 바꿔 주는 것이 **소프트맥스(softmax)** 예요.
>
> 실제 학습에서는 이 점수 z를 사람이 아니라 가중치가 만들고, 가중치는 A에서 본 경사하강법으로 고쳐져요.

**보너스 관찰 (작게)**:
> 세 슬라이더를 *같은 양만큼* 올려 보세요 — 막대가 변하지 않아요. 차이만 중요해요.

**하단 한 줄**:
> 정답 클래스의 막대를 1쪽으로 끌어올릴수록 작아지는 손실이 **교차 엔트로피(cross-entropy)** 예요 — 회귀에서 MSE가 했던 일을, 분류에서는 이 손실이 맡습니다.

#### 5) C1 — 오답 예시 박스 (혼동 행렬 표 만들지 말 것)

> 정확도 숫자만 보면 모델이 **어떤 종류를 헷갈리는지**는 알 수 없어요. 클래스별로 모델이 틀린 그림을 모아 보면, 모델의 약점이 한눈에 보입니다.

- 인터랙션: 클래스 칩 가로 줄 → 클릭 시 *그 클래스인데 다른 클래스로 답한* 도트들 그리드
- 각 도트 아래 한 줄: `정답: 동그라미 → 답: 세모`
- 컴포넌트 재사용: Phase12 `SampleGallery`의 "틀린 것만 보기" 모드 + 클래스 필터

#### 6) C3 — 역전파 박스

> 은닉층이 생기면 가중치가 여러 층에 흩어져 있어, 어느 가중치를 얼마나 고쳐야 할지 한눈에 알기 어려워요.
>
> 출력층의 오차는 **A4에서 본 그대로** (예측 − 정답)예요. 은닉층은 자기 정답이 없지만, *바로 뒷층의 오차*를 받아 자기 몫으로 나눠 가집니다 — 그래서 화살이 **거꾸로** 흘러요. 이 거꾸로 흐름이 **역전파(backpropagation)** 예요.
>
> 각 가중치는 A4에서 본 `dw = 평균(e·x)` 모양의 식을 자기 층 입력 x로 똑같이 써서 갱신돼요. **A5에서 본 한 step의 갱신식이 모든 층에 동시에 적용**된다고 보면 됩니다.

#### 7) C4 — 3출력 → 10출력 다리 (도입부 한 문단)

> B5에서는 출력 뉴런 3개로 동그라미·세모·네모를 갈랐죠. MNIST는 입력이 **28×28로 훨씬 크고**, 출력 뉴런을 **10개**로 늘려 0~9 숫자를 가릅니다. **softmax로 확률을 만든다는 출력층의 원리는 같지만**, 입력이 커진 만큼 **C3에서 본 것처럼 은닉층 크기도 함께 키워야** 해요.

### 10-2. 시각화 설계 2건

#### V1. softmax 경쟁 막대 (B5)

- **컴포넌트 재사용**: Phase10의 `Bar`, `WeightVis` 색 어휘
- **구성**: 출력 뉴런 3개(동그라미/세모/네모) 옆 수평 막대 3개 + z 슬라이더 3개 (`−5 ~ +5`)
- **고정 라벨**: 슬라이더 위 `z = 출력 뉴런이 가중합으로 만든 점수`, 막대 위 `합 = 100%`
- **핵심 인터랙션 2가지**
  1. 한 슬라이더 +3 → 해당 막대 자람 + 다른 두 막대 *지수적으로* 짧아짐 (winner-take-most)
  2. 세 슬라이더 동시 +5 → 막대 변화 없음 (시프트 불변, 보너스 관찰)
- **금지**: softmax 식 전개, "정규화 함수" 같은 표현

#### V2. 역전파 빨간 점선 흐름 (C3)

- **컴포넌트 재사용**: Phase11 `NetworkDiagram` (입력 4 / 은닉 3 / 출력 2 작은 망) + Phase5 `backColor` (`rgb(190,18,60)`) + `strokeDasharray="7 5"` + `nv-backflow` 펄스(1.1s)
- **레이어 구조**
  - 정방향: 회색 얇은 간선, 왼 → 오 (기존 그대로)
  - 역방향: 빨간 점선 곡선, 출력 → 은닉 → 입력. 화살촉 입력 쪽
- **인터랙션**: "한 step 진행" 버튼 → 0.3초 시차 펄스. 각 층 가중치 라벨이 도달 시 깜빡임
- **금지**: 체인룰 수식, ∂ 기호 노출
