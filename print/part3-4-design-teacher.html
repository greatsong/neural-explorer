3·4부 — 직접 만들기와 깊은 학습

앱 페이즈 10~12 · 학생용과 동일 구조 + 정답·해설
핵심 메시지 "복잡한 문제는 더 큰 신경망을 요구한다, 그러나 크기만 키우면 과적합이 늘어난다." 3·4부에서는 모델 크기·데이터 양·증강 세 축으로 일반화 감각을 길러내는 게 목표.

STEP 1. 도트 그림 학습 — 가중치 시각화의 의미

활동지 운영학생용은 두 라벨 평균 패턴(가로줄 A·세로줄 B)을 미리 그려 제공한다. 학생들이 직접 도트 캔버스에 그리지 않고, 두 평균을 머릿속으로 빼서 가중치 시각화 결과를 칸별로 채우는 활동.
1-1두 라벨 패턴 — 운영
수업 팁"위쪽 가로줄(A) vs 왼쪽 세로줄(B)"은 대조가 강하고 겹치는 영역이 작은 의도적 단순 사례. 시간 여유가 있으면 다른 페어("원 vs X" 등)를 추가로 제시해 같은 분석을 반복.
1-2가중치 시각화 — 정답
정답 — 8×8 칸별 분류 위쪽 두 줄(0·1행) 중 왼쪽 두 칸(0·1열) 4개 = 둘 다 켜짐 → 0(점)
위쪽 두 줄의 나머지 12칸(0·1행 × 2~7열) = A에만 켜짐 → 양수(흰색)
왼쪽 두 열의 나머지 12칸(2~7행 × 0·1열) = B에만 켜짐 → 음수(검정)
그 외 36칸 = 둘 다 안 켜짐 → 0(빈칸)
한 줄 의미"가중치 시각화 = 두 라벨의 평균 픽셀 차이가 된다."
자주 틀림겹치는 4칸을 양수로 채우는 경우. "둘 다 켜진 자리는 분류에 도움 안 됨 → 0"임을 짚을 것.
1-3새 그림 분류 — 정답
정답 "위쪽 두 줄만 켜진 그림": A 양수 영역에 픽셀이 모임 → 점수 부호 = + → 예측 = A.
"왼쪽 두 줄만 켜진 그림": B 음수 영역에 픽셀이 모임 → 점수 부호 = − → 예측 = B.
한 줄 정의"단일 뉴런이 두 그림을 구분한다 = 두 라벨의 평균 차이 패턴과 입력의 내적을 보고, 양수면 A·음수면 B로 결정하는 일." 학생 답이 "비슷한 무늬가 그림을 구분한다"는 직관 수준이어도 인정.
1-4학습 데이터 양과 일반화 — 정답
정답 테스트 정확도는 올라간다 (60→80→92%).
학습 정확도와 테스트 정확도의 차이는 줄어든다 (40%p → 20%p → 4%p).
이 현상의 이름: 과적합(overfitting) 해소 / 또는 일반화 향상. (2부 STEP 2에서 본 개념의 반복.)
수업 팁"학습 정확도가 100% → 96%로 떨어진 게 좋은 신호"라는 점을 다시 한 번 짚어주면, 2부에서 본 일반화 개념이 픽셀 분류에서도 그대로 적용됨이 명확해짐.

보충 활동 ★. 평균 그림 빼기 — 가중치 색지도 예측 (정답)

활동 의도학습 후 앱이 보여주는 "가중치 색지도"가 두 라벨의 평균 그림 차이와 닮아 있다는 사실을 학생이 손으로 먼저 예측하고, 앱과 비교하게 한다. "신경망이 무엇을 본 것인가"를 시각적으로 잡는 활동.
★1·★2평균 격자 만들기 — 운영 정답
예시 정답 (라벨 ●(채움 동그라미) vs 라벨 ○(빈 동그라미))
  • 평균 ● 격자: 동그라미의 가장자리 8칸은 5에 가깝고(거의 모두 칠해짐), 안쪽 칸도 4~5에 가깝다(채움형이라 안쪽도 칠해짐).
  • 평균 ○ 격자: 가장자리 8칸은 5에 가깝지만, 안쪽 칸은 0~1에 가깝다(빈 동그라미라 안쪽이 거의 비었음).
  • 차이 격자(A − B): 가장자리는 0 근처(서로 비슷), 안쪽 칸은 +4 근처로 큰 양수가 나옴 → 진한 파랑.
수업 팁차이가 0 근처인 가장자리 칸은 "두 라벨이 똑같이 갖는 신호"라 분류에 도움이 안 된다는 점을 짚어 주면, "신경망이 의미 있는 차이만 가중치로 학습한다"는 한 줄 통찰로 이어짐.
★3앱과 비교 — 일치도 패턴
예시 정답대부분 학생은 ② 부분적으로 비슷하다로 답함. 안쪽 칸은 학생 예측대로 큰 양수가 나오지만, 가장자리 일부 칸은 시그모이드의 비선형성과 24장이라는 작은 데이터 양 때문에 예상과 어긋날 수 있음. 결론 한 줄 예시: "두 라벨의 평균 그림 차이를 학습이 자동으로 찾아낸 결과"라고 답하면 만점.
자주 틀림① "완전히 같다"고 답 — 시그모이드 출력의 0~1 짓누름 효과를 무시한 답. ② "전혀 다르다"고 답 — 24장이라는 작은 데이터로 인한 잡음을 패턴 자체의 어긋남으로 오해. 학습 데이터를 더 많이 모으면 일치도가 ②→①로 옮겨감을 함께 안내할 것.
전이이 활동이 P10 10.4 "다양성이 곧 일반화 능력"의 직관 근거가 됨. 평균 그림이 잘 잡히려면 한 사람의 그림이 아니라 여러 사람의 그림이 필요하다는 점이 자연스럽게 도출됨.

STEP 2. 신경망 설계 — 단계 A·B·C 비교 (Phase 11)

앱 운영Phase 11은 난이도 단계 3개(🟢 A=2종 / 🟡 B=3종 / 🔴 C=10종)를 차례로 학습시키며 비교표를 채우는 구조. 단계마다 추천 은닉 크기가 다름(A=4, B=8, C=16). 활동지 표는 셋 다 은닉 8로 통일해 클래스 수 변화의 영향만 따로 보게 만든 의도적 단순화. 학생이 앱에서 추천값으로 다시 학습시키면 정확도가 더 오르는 모습(특히 C)을 추가로 관찰하게 안내할 것. 학습 식 (페이즈 5의 갱신식 w ← w − η · dw)이 다층에서도 그대로 적용된다는 점이 페이지 상단 박스에 명시돼 있음.
2-1단계 A·B·C — 정답
정답 클래스 수가 늘어날수록 정확도는 내려간다.
파라미터 수가 비슷한데 정확도 차이가 큰 이유: 클래스 수가 많을수록 모델이 어려워진다(같은 표현력으로 더 많은 결정 경계를 만들어야 함).
단계 C 검산: (784+1)·8 + (8+1)·10 = 6280 + 90 = 6,370.
수업 팁"같은 신경망에 더 어려운 문제를 던지면 정확도가 떨어진다"는 직관이 핵심. 다음 2-2에서 "그럼 신경망을 키우면?"으로 자연스럽게 이어진다. 도전 활동: 앱에서 단계 C에 은닉 16(추천값)·24·32로 학습시켜 정확도 향상을 직접 관찰.
2-2은닉 크기 — 정답
정답 은닉을 키우면 정확도는 올라간다(70→82→88→91%).
하지만 두 배씩 키워도 증가 폭(12 → 6 → 3)은 점점 작아진다(수확체감).
이유 작은 모델이 못 잡던 패턴을 큰 모델이 잡으면서 정확도가 빠르게 오르지만, 이미 잡힌 패턴 위에 더 큰 모델을 얹어도 새로 잡을 패턴이 적기 때문. 또한 과적합이 시작되어 학습 정확도만 오르고 시험 정확도는 멈출 수 있음.
자주 틀림"키우면 무조건 좋다"는 답. "증가 폭이 줄어든다"는 점과 과적합 위험을 함께 짚을 것.
2-31·7 혼동 — 모범 답안
예시1과 7은 모두 세로 직선이 핵심 특징이다. 7은 위쪽에 짧은 가로획이 추가될 뿐이라, 가로획이 흐릿하거나 작은 글씨에서 신경망이 두 클래스의 신호를 분리하기 어렵다.

STEP 3. MNIST 도전

앱 운영Phase 12에는 5단계 프리셋(🐣 시작·🌱 작게·🌳 보통·🚀 크게·👑 최강) 버튼이 있다. 학생이 한 번 클릭하면 은닉층 구성 + 에폭이 동시에 셋업되는 구조. 활동지 표는 위쪽 4단계에 매칭. 👑 최강(128·64 2층·30에폭)은 도전 과제로 남겨두고, 학생들이 시험 정확도가 더 오르는지 / 아니면 학습-시험 격차만 더 벌어지는지(과적합) 직접 비교하게 운영. 에폭 슬라이더는 1~40까지 자유롭게 조정 가능하니 프리셋 외 임의 설정도 가능.
3-1구조·설정 변화 — 정답
정답 🐣 → 🌱 은닉 4 → 8로 키우면 시험 정확도 오른다(65 → 78%).
🌱 → 🌳 은닉 8 → 32, 에폭 8 → 15로 키우면 오른다(78 → 88%).
🌳 → 🚀 1층 [32]에서 2층 [64, 32]로 깊게 쌓으면 오른다(88 → 92%).
🚀에서 학습 97%·시험 92%로 격차가 커지는 현상: 과적합(overfitting).
수업 팁🐣 → 🌱·🌳 단계는 "표현력 부족" 해소, 🚀 → 👑 단계는 "표현력 과잉" 시작. 👑 최강을 직접 돌려본 학생이 시험 정확도가 92~93%에서 멈추거나 오히려 떨어지는 모습을 발견하면 매우 좋은 신호. "무한정 키우면 학습 정확도만 99%로 가고 시험 정확도는 천장에 부딪힌다"를 자기 말로 정리시킬 것.
3-2증강의 효과 — 모범 답안
예시증강 전: 학생이 그린 7이 살짝 기울어져 모델이 1로 오인. 증강 후(같은 7을 회전·이동·축소 15번): 모델이 "이 그림은 각도·위치·크기가 변해도 같은 7이다"라는 불변성을 학습 → 적중.
자주 틀림"증강 = 데이터를 늘린 것 자체가 좋다"는 답. "같은 라벨의 다양한 변형을 보게 해 모델이 본질적인 패턴(7의 모양)에 집중하도록 한다"는 점이 핵심.
3-3한계 페어 — 모범 답안
예시 4-9: 둘 다 위쪽에 닫힌 곡선·아래쪽에 세로 직선. 차이는 4의 위쪽 곡선이 한 점에서 살짝 열려 있다는 정도. 글씨 두께·기울기에 따라 그 한 점이 흐려지면 구별 불가.
3-8: 두 곡선이 닫힘 vs 열림. 8의 두 원이 모두 닫혀 있고 3은 오른쪽만 곡선. 픽셀 단위로는 한두 칸 차이.
5-6: 위쪽 가로획·중간 곡선이 거의 같고, 차이는 5의 위쪽이 직각인지 둥근지.
전이이런 픽셀 단위 미세 차이는 단순 MLP가 잘 못 잡고, 지역 특징을 잡는 구조(CNN)가 필요하다는 동기부여로 연결. 관심 있는 학생에게 한 마디 짚어줄 만함.

보충 활동 ★. 파라미터 예산 챌린지 — 1만 개 이하 (정답)

활동 의도"무작정 키우기" 대신 "예산 안에서 잘 쓰기"를 한 번 직접 경험. 같은 1만 개 이하라도 학습률·에폭에 따라 정확도 차이가 큼을 보여 주면, 모델 크기와 학습 조건이 별개임을 자연스럽게 잡음.

A 도달 두 라벨 데이터를 5장 이상 모아 95% 이상 도달. 가중치 시각화의 의미를 자기 말로 설명. A·B·C 모두 학습 후 클래스 수와 정확도의 관계를 그래프적 직관으로 설명. 1·7 혼동 원인을 픽셀 패턴으로 설명. 은닉/에폭 변화에 따른 학습 vs 시험 차이를 관찰해 과적합 신호를 식별. 증강의 효과를 "불변성 학습"으로 설명.
B 진행 적당히 학습은 됐지만 가중치 시각화 의미를 직관 수준으로만 설명. 두 단계만 비교, 클래스 수의 영향을 부분적으로만 설명. 모델 변경의 영향을 표로 기록하지만 일반화 차이를 못 짚음.
C 보충 데이터 부족으로 학습 안 됨. 가중치 시각화 보고 답을 적지 못함. 한 단계만 풀고 비교 못함. 설정을 바꿨지만 변화 의미를 해석하지 못함.
전이 포인트4부까지는 모두 분류 문제였다. 5부에서는 같은 도구로 새로운 것을 만드는(생성) 모델로 넘어간다. "분류기를 뒤집으면 생성기가 된다"는 직관을 미리 던져둘 것.
