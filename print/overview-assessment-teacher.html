종합 평가 — 채점 가이드

학생용과 동일 문항 순서 + 정답 + 부분점수 규칙 + 자주 틀리는 지점
채점 원칙 ① 객관식은 1답만 정답. ② 단답·계산은 정답 그대로면 만점. ③ 서술은 키워드 충족 여부로 부분점수. ④ 계산 도중 절차가 옳고 마지막 산수만 틀린 경우 1~2점 부분점수.

① 1부 — 뉴런의 기초 (16점)

1인공 뉴런의 부품3점
정답① 가중치 (w)   ② 편향 (b)   ③ 활성화 함수 (예: ReLU)
부분점수각 1점. ③에서 "ReLU"라고 함수명만 적어도 인정.
2순전파 계산4점
정답z = (−2)·4 + 1·3 + 5 = −8 + 3 + 5 = 0  /  y = ReLU(0) = 0
부분점수z 값 정답 2점, y 값 정답 2점. z가 −8+3+5 같이 풀이는 옳고 산수만 틀리면 1점.
자주 틀림−2·4를 −24로 잘못 계산. ReLU(0)을 1이나 −0으로 답.
3오차와 손실3점
정답L = (5 − 8)² = 9
제곱 3가지 이유① 부호 상쇄 방지 — 어떤 점은 +3, 어떤 점은 −3으로 어긋나 있을 때 그대로 더하면 0이 되어 "오차 없음"처럼 보임. 제곱하면 모두 양수.
② 큰 오차에 더 큰 페널티 — 1²=1 vs 10²=100. 모델이 큰 실수를 먼저 줄이려고 함.
③ 매끄러운 곡선(미분 가능) — 손실이 부드러운 포물선이 되어 자동 학습이 "어느 쪽이 내리막인지"를 정확히 계산할 수 있음.
부분점수L 값 1점 + 이유 3가지 중 2개 정답이면 2점, 3개 모두면 만점(3점). 한 가지만 적었으면 0.5점.
4한 step 자동 갱신 (ReLU 모델)6점
정답 z = 1·2 + 0 = 2  /  ŷ = ReLU(2) = 2 (z > 0이므로 그대로 통과)
e = 2 − 5 = −3
w 변화량 = e · x = (−3)·2 = −6  /  b 변화량 = e = −3
w ← 1 − 0.1·(−6) = 1.6
b ← 0 − 0.1·(−3) = 0.3
부분점수z·ŷ·e·w변화량·b변화량·w·b의 7항목 중 6개 이상 정답이면 만점, 5개 5점, 4개 4점 ... 부호 약속이 일관되면(예: e = y − ŷ로 정의) 그 정의에 따라 풀었으면 인정.
자주 틀림① ReLU 단계를 빼먹고 z를 그대로 ŷ로 쓰면 결과 동일 → 인정 가능. 다만 "z > 0이면 ReLU(z)=z"임을 명시해야 만점.
② 업데이트 식에서 마이너스 빼먹어 발산 방향(w → 0.4)으로 내려간 답.
참고z < 0인 경우라면 ReLU(z) = 0이 되어 ŷ = 0, e = −5. 이 경우 ReLU'(z) = 0이라 변화량도 모두 0 → 학습이 멈춤(dead ReLU). 이번 문제는 z > 0이라 그 함정에 걸리지 않는다는 점을 짚어둘 것.

② 2부 — 분류와 평가 (18점)

5혼동행렬 지표4점
정답 정확도 = (35 + 50)/100 = 85%
정밀도 = 35/(35 + 10) = 35/45 ≈ 0.778 (또는 7/9, 77.8%)
재현율 = 35/(35 + 5) = 35/40 = 0.875 (87.5%)
부분점수각 1점 + 분모 정확히 쓴 식 1점.
6객관식3점
정답③ 정확도 99%, 재현율 0%
자주 틀림② 선택 — 정확도는 맞으나 재현율 100%로 헷갈림. TP=0이라 분자가 0임을 짚을 것.
7지표 매칭6점
정답 스팸 메일: 정밀도 — 정상 메일을 스팸으로 잘못 분류하면(FP) 사용자가 중요한 메일 놓침. FP 비용↑.
암 검진: 재현율 — 진짜 환자를 놓치면(FN) 생존율 급락. FN 비용↑.
동전 던지기: 정확도 — 클래스 비율과 두 오류 비용이 같아 단순 정확도가 충분.
부분점수지표 1점 + 이유 1점, 행마다 총 2점. 이유에 "FP/FN 중 어느 쪽이 비싼가" 언급이 있으면 만점.
8학습 vs 시험 정확도5점
키워드"과적합(overfitting)"이라는 용어가 등장하면 2점.
개선 방안 예시① 데이터 추가 ② 데이터 증강 ③ 모델 크기 줄이기 / 정규화 ④ 학습 일찍 멈추기(early stopping) ⑤ 일반화 점검을 위한 검증 셋 사용. 두 가지 이상 적으면 3점.
부분점수과적합 진단 2점 + 방안 1개 1점, 2개 이상 3점.

③ 3·4부 — 도트 학습과 깊은 학습 (14점)

9은닉층의 역할4점
키워드"입력의 부분 패턴(특징)을 추출", "출력층이 그 특징들을 조합해 분류"라는 두 단어가 모두 들어가면 4점.
부분점수특징 추출 2점 + 조합/판단 2점. "비선형성을 더한다" 같은 답도 부분 인정(2점).
10객관식 — 데이터 증강3점
정답② 같은 라벨의 다양한 변형을 보게 해 위치·각도·크기에 강해진다
자주 틀림① 선택 — "양이 늘면 무조건 좋다"는 단순화는 오답.
11클래스 수와 신경망 크기4점
예시 답안"같은 구조라면 클래스가 많아질수록 시험 정확도가 떨어진다. 10종을 잘 분류하려면 더 큰 은닉층(또는 더 깊은 층)이 필요하다. 2종(0/1)은 작은 신경망으로도 99% 가능하지만 10종은 더 풍부한 표현력을 요구한다."
부분점수"정확도 차이" 2점 + "표현력/모델 크기 필요" 2점. 한쪽만 적으면 2점.
12파라미터 수3점
정답(64+1)·8 + (8+1)·2 = 65·8 + 9·2 = 520 + 18 = 538개
부분점수편향 빼먹어 64·8 + 8·2 = 528로 답하면 2점. 식 자체는 옳지만 산수 틀림 → 2점.

④ 5부 — 생성으로 (12점)

13분류 vs 생성4점
정답 분류: 입력 = 데이터(그림 등), 출력 = 라벨
생성: 입력 = 잠재 좌표 또는 노이즈, 출력 = 데이터(그림 등)
부분점수각 1점.
14오토인코더 구조4점
정답 ① 인코더 — 입력을 짧은 잠재 벡터로 압축
② 잠재(코드, latent z) — 핵심 정보를 담은 저차원 벡터
③ 디코더 — 잠재 벡터를 다시 입력 크기의 출력으로 복원
부분점수이름과 설명 각 0.5점씩, 항목당 1점, 전체 4점 만점에서 가깝게 차감.
15잠재 공간 외 영역4점
정답 키워드"이상하거나 깨진 그림이 나옴 / 어느 라벨에도 속하지 않음" + 이유는 "그 좌표는 학습 데이터가 분포하지 않아 디코더가 그 입력을 본 적이 없기 때문".
부분점수현상 2점 + 이유 2점.

⑤ 6부 전반 — 글자에서 의미로 (14점)

16한글 토큰이 많은 이유4점
정답 키워드① 학습 코퍼스에 한글 텍스트가 영어보다 적음 ② 자주 등장하는 한글 음절 페어가 적게 학습됨 ③ 결과적으로 한글은 글자 단위 또는 바이트 단위로 더 잘게 쪼개진다.
부분점수코퍼스 빈도 차이 2점 + 결과(쪼개짐) 2점. "한글이 복잡해서"만 적으면 1점.
17객관식 — 원-핫 한계3점
정답③ "학습할 파라미터가 너무 많아 학습이 느리다" — 원-핫 자체에는 학습 파라미터가 없음. ①②④는 모두 진짜 한계.
자주 틀림"옳지 않은"을 놓치고 ②(cos = 0)를 답하는 경우.
18임베딩 룩업3점
정답W = 1000 × 64 = 64,000개
부분점수크기 1점 + 곱셈 1점 + 결과 1점.
19의미 산수의 정체4점
예시 답안"임베딩 공간에서 두 단어를 잇는 화살표는 두 단어의 의미 차이를 가리킨다. king→queen 화살표는 '왕족이라는 속성을 유지한 채 성별만 남→여로 바꾸는 변환'이다. 같은 변환을 다른 단어(father)에 적용하면 같은 방향의 결과(mother)가 나온다. 즉 의미가 위치가 아니라 벡터의 방향으로 표현되기 때문."
부분점수"화살표 = 의미 차이"라는 직관 2점 + "같은 변환은 다른 단어에서도 평행" 2점.

⑥ 6부 후반 — 시퀀스에서 GPT까지 (18점)

20평균 임베딩 한계3점
예시 답안"단어 임베딩의 단순 평균은 순서 정보를 잃는다. 그래서 모델이 의미를 보존하려면 순서를 다루는 별도 메커니즘(RNN·어텐션 등)이 필요하다."
부분점수"순서 정보 상실" 2점 + "별도 메커니즘 필요" 1점.
21어텐션 점수5점
정답 S(a, b) = 1·2 + 0·1 + 2·0 = 2
S(a, c) = 1·1 + 0·1 + 2·1 = 3
softmax 가중치는 c 쪽이 더 큼 (점수가 큰 쪽이 가중치도 큼).
부분점수S(a,b) 2점 + S(a,c) 2점 + 비교 1점.
22가중합3점
정답 O = 0.6·(2, 1, 0) + 0.4·(1, 1, 1)
   = (1.2, 0.6, 0) + (0.4, 0.4, 0.4)
   = (1.6, 1.0, 0.4)
부분점수차원별 1점 (3개 차원).
23트랜스포머 블록 부품4점
정답① 멀티헤드 어텐션   ② 잔차 연결 (residual)   ③ FFN (feed-forward, 작은 MLP)   ④ LayerNorm (층 정규화)
부분점수각 1점. 순서·표기 차이 인정. "Self-attention"도 ①의 동의어로 인정.
24객관식 — 온도3점
정답② 분포가 더 평탄해지고 결과의 다양성·의외성이 커짐
자주 틀림① 선택 — T 작을 때와 클 때를 반대로 이해.

⑦ 통합 서술 (8점)

2522단계 한 호흡 정리8점
모범 답안 예시 "단일 뉴런은 입력에 가중치를 곱하고 편향을 더해 출력하며, 정답과의 차이로 손실을 측정해 그 방향으로 가중치를 조금씩 옮기는 학습을 한다. 이렇게 학습된 모델이 새 데이터에서도 잘 맞히는 능력을 일반화라 부른다. 입력이 픽셀로 풍부해지면 단일 뉴런만으로 부족하므로 은닉층을 두고 부분 패턴을 잡아 조합한다. 분류만이 아니라 잠재 좌표에서 그림을 만들어내는 생성 모델로도 같은 도구를 쓸 수 있다. 텍스트는 토큰으로 쪼갠 뒤 임베딩 벡터로 바꾸는데, 이 공간에서는 의미가 위치가 아니라 방향으로 표현된다. 단어들의 관계를 모델이 직접 정하게 하는 장치가 어텐션이고, 이 어텐션과 작은 MLP를 한 블록으로 묶어 깊게 쌓은 것이 트랜스포머다. GPT는 트랜스포머의 마지막 출력에서 어휘에 대한 분포를 만들고 한 토큰을 샘플링해 입력 끝에 붙이는 일을 반복하며 문장을 완성한다."
채점 규칙키워드 8개(가중치·손실·일반화·은닉층·생성·임베딩·어텐션·샘플링) 각 1점. 단순 단어 등장이 아니라 문맥에 맞게 사용되어야 인정. 단어가 등장하지만 의미가 어긋나면 0.5점 차감. 8문장 미만이면 최대 6점, 12문장 초과는 가독성 평가.
자주 틀림키워드를 단순 나열한 답("가중치·손실·일반화 ...에 대해 배웠다") → 0~2점. 4부까지만 정리하고 5·6부 누락 → 최대 4점.

채점 후 점검 — 영역별 진단 가이드

영역득점 12점 미만일 때 짚을 부분
① 1부가중치·편향·활성화의 식 표현(z = wx+b, ReLU)을 다시 시범. 부호 처리 연습 추가.
② 2부혼동행렬 분모 오류가 흔함. TP/FP/FN/TN을 표로 그려 분모 계산을 한 번 더.
③ 3·4부파라미터 계산에서 편향 누락 흔함. (in+1)·out 공식을 칠판에 고정 표기.
④ 5부"분류 ↔ 생성"의 입출력이 뒤집힘을 못 잡으면 보충 활동.
⑤ 6부전반임베딩이 룩업이라는 점을 못 잡았으면 W 그림으로 다시 시범.
⑥ 6부후반어텐션 가중합 계산이 약하면 차원별 분리 표기를 한 번 더 연습.
⑦ 통합키워드 8개 중 6개 미만 등장 시, 각 부의 한 줄 요약 카드를 만들어 다시 묶기.
총평 가이드 80점 이상 = A 도달 / 60~79점 = B 진행 / 60점 미만 = C 보충. 영역별 점수가 평균 대비 ±5점 이상 벌어지면 그 영역의 워크시트(1~6번)를 부분 재학습 권장.
