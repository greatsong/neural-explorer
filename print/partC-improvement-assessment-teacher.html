C — 딥러닝 평가 [정답·해설]

역전파 한 사이클 + 사슬규칙 식 유도 + MNIST·증강 · 채점 기준 · 부분 점수 가이드
앱 페이즈 C1~C3 · 대상: 채점 교사용 · 권장 채점 시간 5~7분/장
평가지의 모든 문제를 번호 그대로 다룹니다. 각 문제마다 ① 채워진 정답, ② 해설(왜 그 답인지·흔한 실수), ③ 채점 기준, ④ 부분 점수 가이드를 묶어 제시합니다.
평가지 데이터 (확인용)
  • x = 3, y = 7 (정답 직선 ŷ = 2x + 1)
  • 초기 가중치: w₁ = 0.4, b₁ = 0.2, w₂ = 1.0, b₂ = 0.5
  • 학습률 η = 0.05
  • 활성화: 뉴런 1 = ReLU, 뉴런 2 = 선형

제1부. 모델 구조와 한 사이클 손계산

문제 1모델 구조 — 라벨 채우기 [정답]
정답
  • 화살표 라벨: x → (w₁) → Σ (+ b₁) → ReLU → (w₂) → Σ (+ b₂) → ŷ
  • 노드 이름: z₁ = w₁·x + b₁,   h = ReLU(z₁),   z₂ = w₂·h + b₂
  • 활성화 종류: 뉴런 1 = ReLU, 뉴런 2 = 선형(없음, 그대로 출력)
  • 모델 식: ŷ = w₂ · ReLU(w₁·x + b₁) + b₂
  • "깊다"의 까닭: 곱·합·활성화 한 묶음을 2번 적용해서, 뉴런이 직렬로 2개 이어져 있기 때문.
해설 — 활동지 0-1과 같은 구조이지만, 활성화 라벨을 학생이 직접 채워야 한다는 점이 차이. 출력층에 ReLU를 잘못 끼워 넣는 학생이 종종 있다(회귀이므로 출력은 선형).
채점 — 가중치/편향 라벨 4개 정확(2점), 노드 이름 z₁·h·z₂ 정확(1점), 활성화 종류 정확(1점), 모델 식 정확(1점). 총 5점.
흔한 실수 — 출력에 ReLU를 끼움 / w와 b를 같은 화살표 위에 묶어 적음 / h와 z₂를 혼동.
문제 2순전파 4단계 [정답]
① z₁ = 0.4 · 3 + 0.2 = 1.4
② h = ReLU(1.4) = max(0, 1.4) = 1.4
③ z₂ = 1.0 · 1.4 + 0.5 = 1.4 + 0.5 = 1.9
④ ŷ = z₂ = 1.9
관찰 — z₁은 양수. ReLU′(z₁) = 1. 길이 열려 있다.
해설 — 산술 실수가 가장 흔함. z₁ = 1.4 (양수)이므로 ReLU는 그대로 통과. 만약 z₁이 음수로 나왔다면 학생이 산술을 잘못한 것이거나, w₁·b₁ 자리를 바꿔 적은 것.
채점 — 4값(z₁, h, z₂, ŷ) 각 1점, 관찰(부호·ReLU′·열림 여부) 1점. 총 5점. 산술 실수는 부분 점수(식이 맞고 결과만 틀리면 절반).
문제 3오차와 손실 [정답]
e = ŷ − y = 1.9 − 7 = −5.1
손실 ½·e² = ½ · (−5.1)² = ½ · 26.01 = 13.005
e의 부호 = 음수. 한 줄 의미 — 모델이 정답보다 작게 예측 중. e의 부호가 결정하는 것 = 방향(크기는 |e|가 결정).
해설 — 부호 규약은 e = ŷ − y. 학생이 y − ŷ으로 바꿔 적으면 부호가 뒤집혀 이후 모든 갱신 방향이 반대가 됨 — 이 경우 일관되게 적용했으면 결과 자체는 같으나 채점 기준에선 식 정의를 따른다.
채점 — e 값(1점), 손실 값(1점), 부호 해석(1점), 방향/크기 구분(1점). 총 4점.
문제 4출력 쪽 기울기 [정답]
dw₂ = e · h = (−5.1) · (1.4) = −7.14
db₂ = e · 1 = −5.1
왜 e가 그대로 들어가는가 — 뉴런 2는 출력 바로 앞 층이라 자기 오차가 곧 출력 오차 e이고, 그 층의 입력은 h이므로 dw = (그 층의 오차)·(그 층의 입력) = e·h가 된다.
채점 — dw₂ 값(1점), db₂ 값(1점), 서술 한 줄(1점). 총 3점. 서술에서 "그 층의 입력 = h" 또는 "단일 뉴런 식 dw=e·x의 x 자리에 h가 들어감"이 보이면 만점.
문제 5거꾸로 흐른 신호 — w₂ 거꾸로 → ReLU 거꾸로 ★ [정답]
5-a. 가중치 w₂ 거꾸로 통과 — h의 오차
e_h = e · w₂ = (−5.1) · (1.0) = −5.1  ← h의 오차
5-b. ReLU 문지기 거꾸로 통과 — z₁의 오차
ReLU′(z₁) = 1  (z₁ = 1.4 > 0이므로)
e_z₁ = e_h · ReLU′(z₁) = (−5.1) · (1) = −5.1  ← z₁의 오차
두 값 비교 — e_h = e_z₁ = −5.1, 같다. 까닭: ReLU′(z₁) = 1 (문지기 열림). z₁이 살아 있어 e_h에 1이 곱해진 것이 곧 e_z₁이 됨. 그래도 ReLU′ 자리는 식에서 사라지지 않고 "z₁이 살아 있다"는 사실을 기록한다.
ReLU′ 의미 한 줄 — z₁이 음수일 때 ReLU′ = 0이 되어 거꾸로 가는 길이 막히고, 그 결과 e_z₁ = 0이 되어 뉴런 1의 가중치는 한 사이클 동안 움직이지 않는다(문지기 역할). 이때 e_h(=e·w₂) 자체는 살아 있지만 ReLU 문지기에서 차단됨.
해설 — 이 문제가 역전파의 핵심. 이번 평가지는 w₂=1이라 e_h=e도 우연히 같고, ReLU′=1이라 e_z₁=e_h도 우연히 같다 — 즉 "두 번 우연"이지만 통과 단계 자체는 식에 명시되어야 함을 학생이 이해해야 한다. 일반적으로 |e_h|는 |e|와 다를 수 있고, |e_z₁|는 |e_h|와 다를 수 있음.
채점 — 5-a (e_h 값 2점), 5-b (ReLU′ 값 1점 + e_z₁ 값 2점), 두 값이 같은 까닭(2점), ReLU′ 의미 서술(2점). 총 9점. 핵심 문제이므로 가중치 높음.
흔한 실수 — ReLU′ 단계를 빼고 e_h만으로 dw₁ = e_h·x를 적음 / ReLU′(z₁)을 ReLU(z₁) 그대로 적음(=h) / 두 값이 같다는 점에서 "ReLU 단계가 의미 없다"고 결론(틀림 — z₁≤0이면 막힌다는 사실이 식에 기록됨).
문제 6입력 쪽 기울기 [정답]
dw₁ = e_z₁ · x = (−5.1) · (3) = −15.3
db₁ = e_z₁ · 1 = −5.1
관찰 한 줄 정답 예시 — "d_가중치 = (그 층의 오차) · (그 층의 입력)"이라는 같은 패턴이 뉴런 1·뉴런 2에 모두 적용되며, 뉴런 1에서는 오차 자리에 e_z₁(z₁까지 거꾸로 흘러온 오차), 입력 자리에 x가 들어간다.
해설 — 같은 식의 반복임을 보여주는 자리. 학생이 dw₁ = e·x로 적으면(즉 e_z₁ 대신 e를 그대로 사용) 역전파 핵심을 놓친 것. 큰 감점.
채점 — dw₁ 값(1점), db₁ 값(1점), 패턴 서술(2점). 총 4점.
문제 7갱신 표 [정답]
가중치현재값기울기η · 기울기새 값
w₂1.0−7.14−0.3571.357
b₂0.5−5.1−0.2550.755
w₁0.4−15.3−0.7651.165
b₁0.2−5.1−0.2550.455
관찰 — 4개 dw·db가 모두 음수였으므로, 빼기를 거쳐 4개 가중치는 모두 커졌다.
해설 — e가 음수(ŷ < y)였으므로 모델은 ŷ을 키우는 방향으로 움직여야 함. 빼기(w − η·dw)를 거쳐 음수 기울기는 가중치를 커지게 함 → ŷ이 커짐 → 정답 7에 가까워짐.
채점 — 12개 칸 각 0.5점(총 6점) + 관찰 1점. 총 7점. 부호 실수는 칸당 0.25점 부분 점수.
흔한 실수 — η·dw 자리에서 부호 빼먹음 / 새 값 = 현재 + η·dw로 잘못 계산(이러면 손실이 커진다).
문제 8갱신 검증 [정답]
새 z₁ = 1.165 · 3 + 0.455 = 3.495 + 0.455 = 3.95
새 h = ReLU(3.95) = 3.95
새 z₂ = 1.357 · 3.95 + 0.755 ≈ 5.360 + 0.755 = 6.115
새 ŷ = 6.115,   새 e = 6.115 − 7 = −0.885
새 손실 ½·e² = ½ · 0.7832 ≈ 0.392
새 손실 0.392 < 갱신 전 손실 13.005 → 작다. 손실은 줄었다.
해설 — 한 사이클 만에 손실이 약 33배 줄었다(13.005 → 0.392). 학습률 0.05가 이 단순한 모델·데이터에는 충분히 큰 값임을 보여준다. 만약 새 손실이 더 커졌다면 학생의 갱신 표(문제 7) 부호가 틀렸다는 신호.
채점 — 새 z₁·h·z₂·ŷ·e·손실 6값 각 1점(총 6점) + "줄었다" 판정 1점. 총 7점. 산술이 약간 어긋나도 "감소" 결론이 맞으면 절반 이상 인정.
부분 점수 — 1.357·3.95 자리에서 5.36 / 5.360 / 5.36115 등 소수점 처리에 따라 새 손실이 0.39~0.40 사이로 약간 흔들릴 수 있음 — 모두 정답 처리.

제2부. 개념 — 채점 기준 (서술형)

문제 9왜 4개 가중치를 *동시에* 갱신하는가 [모범 답안]
모범 답안
한 사이클에서 dw₂, db₂, dw₁, db₁ 네 기울기는 모두 같은 시점의 같은 ŷ과 같은 e에서 출발한다. 한쪽 가중치를 먼저 갱신하면 다음 가중치를 구할 때 쓸 ŷ과 e가 바뀌어 버려, 같은 사이클의 일관된 기울기 정보가 사라진다. 따라서 4개 기울기를 모두 구한 뒤 한 번에 동시에 갱신하고, 그 다음 사이클에서 다시 새 ŷ·새 e를 구해 새 기울기를 계산한다. 이렇게 해야 "한 사이클 = 한 step"이라는 단위가 깨지지 않는다.
채점 기준 (체크리스트)
  • □ "같은 시점의 같은 e"에서 4개 기울기가 나온다는 사실 언급 (2점)
  • □ "차례로 갱신하면 ŷ·e가 바뀐다"는 부작용 설명 (2점)
  • □ "한 사이클" 또는 "한 step" 단위 보존 언급 (1점)
  • □ 4~5문장 분량, 자기 말 (1점)
총 6점. 핵심어(같은 e / 같은 시점 / 동시 / 한 사이클) 4개 중 3개 이상 등장 시 5점 보장.
흔한 미흡 — "그게 빠르니까", "병렬로 처리하려고" 등 효율 관점만 적는 경우는 절반 점수(1~2점)만 인정. 핵심은 "정보 일관성".
문제 10왜 출력 쪽이 *먼저* 계산되는가 [모범 답안]
모범 답안
오차 e는 정답 y와 출력 ŷ을 비교해 만들어지므로, 출력 바로 앞 층(뉴런 2)에서만 e를 직접 쓸 수 있다. 뉴런 1은 자기만의 정답이 없어서, 자기 오차를 알려면 출력의 e를 두 번 거꾸로 통과시켜 받아와야 한다 — ① w₂를 거꾸로 통과해 h의 오차 e_h, ② ReLU 문지기를 거꾸로 통과해 z₁의 오차 e_z₁. e_z₁을 만들려면 먼저 e_h가, e_h를 만들려면 먼저 e가 있어야 한다. 따라서 계산 순서는 출력 쪽 dw₂·db₂ → 거꾸로 흐른 신호 e_h → ReLU 문지기 통과 e_z₁ → 입력 쪽 dw₁·db₁ 순으로 고정된다. 이 "거꾸로" 흐름이 곧 역전파다.
채점 기준 (체크리스트)
  • □ "정답 y는 출력에만 있다"는 비대칭성 언급 (2점)
  • □ "뉴런 1은 자기 정답이 없다" 또는 "오차를 받아와야 한다" 언급 (2점)
  • □ "w₂를 거꾸로 통과" 또는 "역전파" 표현 (1점)
  • □ 4~5문장 분량 (1점)
총 6점.
흔한 미흡 — "그냥 식이 그렇게 생겼으니까"(0점) / "수학적으로 그렇게 정해져 있어서"(0점) — 정답·오차의 비대칭이라는 본질이 빠지면 점수 없음.
문제 11MNIST 분포 불일치 + 증강 [모범 답안]
모범 답안
MNIST의 학습 분포는 28×28로 정렬되고 굵기·기울기가 일정한 손글씨인 반면, 내가 직접 그리는 사용 분포는 280×280에 위치·굵기·기울기가 자유로워 픽셀 단위로 보면 학습 데이터와 크게 다르다. 모델은 학습 분포 안에서만 잘 외운 상태라, 사용 분포의 그림이 들어오면 자신 있게 틀리게 된다. 이 차이를 메우는 도구가 증강으로, 학습 데이터 한 장을 회전·이동·스케일·전단으로 N장으로 늘려 분포의 폭을 넓힌다. 그렇게 다시 학습하면 사용 분포에 가까운 변형도 모델이 같은 숫자로 인식하게 되어 정확도가 올라간다.
채점 기준 (체크리스트)
  • □ "학습 분포" 단어 사용 + 그 의미 설명 (2점)
  • □ "사용 분포" 단어 사용 + 그 차이 설명 (2점)
  • □ "증강" 단어 사용 + 분포의 폭을 넓힌다는 효과 설명 (2점)
  • □ 4~5문장 분량 (1점)
  • □ 세 단어 모두에 밑줄 표시 (1점)
총 8점. 세 단어 중 1개라도 빠지면 최대 5점.
흔한 미흡 — "데이터가 부족해서"만 적음(분포 개념 누락, 2점) / "내가 그림을 못 그려서"(0점, 책임 전가) / 증강을 "데이터 양 늘리기"로만 설명(분포 폭 언급 없으면 1점 감점).

제3부. 심화 — 시나리오 (선택) [정답]

문제 12Dying ReLU [정답]
ReLU(z₁) = h = 0,   ReLU′(z₁) = 0
→ e_h = e · w₂ = (0이 아님 — w₂에 따라 살아 있음)
→ e_z₁ = e_h · ReLU′(z₁) = e_h · (0) = 0  ← 여기서 끊김
→ dw₁ = e_z₁ · x = 0,   db₁ = 0
결과 — 뉴런 1의 가중치(w₁, b₁)는 움직이지 않는다. 이 현상의 이름: Dying ReLU(죽은 ReLU).
위험성 한 줄 — 한 번 z₁이 음수로 잠긴 뉴런은 dw₁·db₁이 0이라 갱신이 멈추고, 다음 사이클에서도 같은 음수 z₁을 만들어내 영원히 회복되지 않을 수 있다(뉴런이 "죽는다").
해설 — h = 0이므로 dw₂ = e·h도 0이 됨에 주의(즉 이 뉴런이 출력에 미치는 통로 자체가 닫힘). db₂ = e는 여전히 0이 아니지만, 뉴런 1 쪽 학습은 완전히 멎는다.
채점 — 5개 0값(1점씩, 총 5점) + 이름(1점) + 위험성 서술(2점). 총 8점. "죽은 ReLU" 한국어, "Dying ReLU" 영어 모두 정답.
문제 13두 번째 사이클의 효과 [정답]
예측 — 작다 (또는 "비슷하다"도 부분 인정).
까닭(모범) — 한 사이클 후 |e|가 5.1에서 0.885로 크게 줄었기 때문에, 두 번째 사이클의 기울기들도 모두 첫 사이클의 약 1/6 수준으로 작아진다. 같은 학습률을 곱해도 가중치 변화 폭이 작아져 손실 감소 폭도 줄어든다. 손실이 0에 가까워질수록 사이클당 감소 폭은 점점 작아지는 수렴 양상을 보인다.
채점 기준
  • □ "작다" 또는 "비슷하다" 예측 (1점)
  • □ |e|가 줄었다는 사실 언급 (2점)
  • □ 기울기 크기 또는 가중치 변화 폭이 작아진다는 연결 (2점)
  • □ 수렴/근접 직관 언급 (1점)
총 6점.
참고 (수치 검증) — 두 번째 사이클: 새 e = −0.885, 새 dw₂ = −0.885 · 3.95 ≈ −3.50, 새 dw₁ = (−0.885 · 1.357 · 1) · 3 ≈ −3.60 등 — 모두 첫 사이클의 약 1/4~1/6. 두 번째 사이클 후 손실은 약 0.005~0.02 수준으로 감소(감소 폭 ≈ 0.39, 첫 사이클 감소 폭 ≈ 12.6의 1/30 수준).
총점·등급 가이드 (참고)
  • 제1부(문제 1~8): 5+5+4+3+9+4+7+7 = 44점 (기능 — 손계산. 문제 5는 ReLU 통과 단계 분리로 9점으로 강화)
  • 제2부(문제 9~11): 6+6+8 = 20점 (개념 — 서술)
  • 제3부(문제 12~13, 선택): 8+6 = 14점 (심화)
  • 합계 78점 / 핵심 64점(제1·2부). 60점 만점 환산 시 제3부는 가산점.
등급 기준(권장) — A: 56점 이상(제1·2부 기준), B: 46~55, C: 36~45, D: 26~35, F: 25 이하. 손계산 부호 실수는 한 번만 감점하고, 같은 패턴의 후속 실수는 부분 점수로 처리해 학생이 "역전파의 흐름(특히 두 번 거꾸로 통과)을 잡았는가"를 우선 평가한다.
교사 팁 — 자주 보이는 오개념 3가지
  1. e_h, e_z₁을 e와 같다고 단순히 외움 — 이번 평가지에선 두 번 우연히(w₂=1로 e_h=e, ReLU′=1로 e_z₁=e_h) 같지만, w₂가 1이 아니거나 ReLU가 막히면 달라진다는 점을 강조.
  2. ReLU 통과 단계를 생략 — e_h만 적고 곧장 dw₁ = e_h·x로 적으면 ReLU 문지기 단계를 식에서 잃은 것. z₁ ≤ 0인 경우를 설명할 수 없게 됨.
  3. "동시 갱신"의 이유를 효율로만 설명 — 핵심은 정보의 일관성(같은 시점의 같은 e). 효율 답안은 절반 점수.
  4. 증강을 "데이터 양 늘리기"로만 이해 — 핵심은 분포의 "폭"을 넓히는 것이지 단순 양 증가가 아니다. 같은 그림 100장 복사해도 효과 없음.
