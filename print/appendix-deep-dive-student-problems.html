TOPIC 1. 활성화 함수의 세계

1주제 한 줄
ReLU 외에 어떤 활성화 함수들이 있고, 각자가 *어떤 문제를 해결*하려고 만들어졌는가?
우리 수업의 어디와 이어지나
  • A1에서 ReLU·sigmoid·linear를 토글로 비교했지만, 같은 z에서 결과가 다르다는 사실까지만 봤음.
  • C1에서 Dying ReLU(z₁ < 0이라 ReLU′ = 0 → e_h = 0)를 손계산으로 발견했지만, 이게 모델 전체에 어떤 영향을 주는지는 안 다뤘음.
탐구 관점 질문
  • sigmoid는 z가 매우 크거나 작을 때 미분이 0에 가까워진다. 깊은 망에서 이게 *어떤 학습 문제*로 이어질까?
  • Leaky ReLU가 음수에도 작은 기울기를 살리는 이유는? Dying ReLU와 어떻게 연결되나?
  • tanh와 sigmoid는 모양이 비슷하지만 출력 범위가 다르다(−1~1 vs 0~1). 어느 쪽이 *0 중심*이라 학습에 유리한 이유는?
  • GELU·Swish 같은 *부드러운 ReLU*가 트랜스포머에서 표준이 된 이유는?
주요 개념 vanishing gradient Dying ReLU Leaky ReLU / PReLU tanh GELU / Swish He / Xavier 초기화 saturation 영역
자료 찾기 — 직접 검색·읽고 비교
  • "Activation functions cheat sheet" 또는 한국어 "활성화 함수 비교"
  • 각 함수의 그래프와 미분 그래프를 *손으로 그려* 비교하면 vanishing gradient가 한눈에 보임
  • 같은 데이터로 sigmoid 깊은 MLP vs ReLU 깊은 MLP를 학습시키는 코드 찾아 직접 돌려 보기
미니 실험 — 직접 그려 보기
sigmoid(z)와 그 미분 σ′(z) = σ(z)(1−σ(z))의 그래프를 −6 ≤ z ≤ 6 범위에서 *손으로 그려* 보세요. 미분 값이 가장 큰 z는? 가장 작은 z는? 깊은 망에서 그라디언트가 *왜 사라지는지* 그래프 한 장으로 설명해 보세요.
내 정리 — 위 질문 4가지에 모두 자기 말로 답한 뒤, *한 문단*으로 "활성화 함수의 세계"를 친구에게 설명한다고 생각하고 적어 보세요.

TOPIC 2. 학습률 스케줄링과 옵티마이저

2주제 한 줄
학습률 η는 *고정된 한 숫자*여야만 하는가? 학습 동안 *계속 조정*하면 어떻게 되는가?
우리 수업의 어디와 이어지나
  • A3에서 η = 1.2 / 1.0 / 0.1 / 0.01 4시나리오로 발산·진동·수렴·느림을 봤음. 그러나 학습 내내 같은 η를 가정.
  • 실제 딥러닝은 학습 초기에 큰 η, 후반에 작은 η가 *최적*인 경우가 많다. 또 가중치마다 *다른 η*를 자동 적용하는 옵티마이저(Adam 등)가 표준.
탐구 관점 질문
  • 학습 초기에 큰 η가 *왜* 유리한가? (큰 보폭이 어디로 데려가는가?)
  • 학습 후반에 작은 η가 *왜* 유리한가? (수렴 점 근처에서 큰 보폭이 일으키는 문제)
  • "warmup"은 처음에 η를 *0에서 키워가는* 방식이다. 왜 그렇게 하는가?
  • Momentum은 *이전 step의 방향*을 일부 기억해 보폭에 더한다. A3 시나리오 중 어느 쪽(진동·발산)에 도움이 될까?
  • Adam은 가중치마다 *과거 그라디언트의 크기*를 누적해 학습률을 자동 조정한다. *자주 변하는 가중치*에는 큰 η, *잠잠한 가중치*에는 작은 η — 이 직관이 학습에 어떤 효과를 주나?
주요 개념 learning rate schedule cosine decay / step decay warmup SGD with momentum Adam / RMSprop learning rate finder adaptive learning rate
자료 찾기
  • "PyTorch lr_scheduler" 공식 문서 — 어떤 스케줄들이 있는지 종류와 그래프
  • "Cyclical Learning Rates for Training Neural Networks" (Smith, 2017) 제목·요약
  • "Adam: A Method for Stochastic Optimization" (Kingma & Ba, 2015) 핵심 식 한 줄
미니 실험 — 작은 SGD 시뮬레이터
A3에서 본 곡선 위에서 momentum을 *직접* 손계산해 보세요. 보폭 식: v ← β·v + dw, w ← w − η·v (β = 0.9). 같은 η에서 진동이 줄어드는지 5 step만 손으로 표 채워 비교.
내 정리 — 같은 모델·같은 데이터에 대해 (a) 고정 η + SGD, (b) 고정 η + momentum, (c) Adam을 학습 곡선으로 비교한다면 무엇이 다를지 자기 말로 예측해 보세요.

TOPIC 3. 교차 엔트로피의 정보이론적 의미

3주제 한 줄
손실이 *왜 하필* −ln(p)인가? 그냥 (1 − p)라든가 다른 함수면 안 되나?
우리 수업의 어디와 이어지나
  • B5 부록 폴드에서 "교차 엔트로피 = 정답 자리 확률을 1쪽으로 끌어올리는 손실 = −ln(p)"까지 봤음. 식의 *모양*만 봤다.
  • 이 식이 *어디에서* 유도되는지(정보 엔트로피·KL divergence·최대 우도)는 다루지 않았다.
탐구 관점 질문
  • "엔트로피"라는 단어는 통계물리·정보이론·머신러닝에서 다 등장한다. *공통된 직관*은 무엇인가? ("불확실성", "놀라움", "정보량"의 관계)
  • 모델 분포 p와 정답 분포 q가 있을 때 *두 분포가 얼마나 다른가*를 재는 KL divergence: D_KL(q || p) = Σ q(x) · log(q(x)/p(x)). 이 식이 어떻게 우리의 −ln(p[정답])로 줄어드나? (정답 라벨이 *원-핫*인 경우)
  • Maximum Likelihood Estimation(MLE) 관점에서, 학습 데이터 N장의 *우도*를 최대화하는 것이 *왜* 평균 −ln(p) 최소화와 같은가?
  • 분류는 CE를 쓰는데 *회귀*는 MSE를 쓴다. 둘은 어떻게 *같은 MLE 가족*에 속하는가? (데이터가 어떤 분포라 가정하느냐의 차이)
주요 개념 정보 엔트로피 KL divergence cross-entropy Maximum Likelihood log-likelihood Bernoulli / Categorical 분포 Gaussian → MSE 유도
자료 찾기
  • "Cross entropy and KL divergence" 또는 한국어 "정보 엔트로피와 교차 엔트로피"
  • 3blue1brown의 "But what is the Central Limit Theorem?" 같은 직관 영상 (엔트로피 직관)
  • "Why MSE for regression and CE for classification" 키워드로 MLE 관점 글 찾기
미니 실험 — 손계산
정답 라벨이 동그라미라 하고, 모델 확률을 다음 두 경우로 두자: (a) [0.9, 0.05, 0.05], (b) [0.5, 0.3, 0.2]. 각각의 CE 손실을 −ln(0.9), −ln(0.5)로 계산하고 *왜 (a)가 더 낮은가* 자기 말로 적기. 단순 정확도(둘 다 동그라미 답이라 100%)로는 못 보는 차이가 무엇인지.
내 정리 — "왜 −ln인가?" 한 질문에 *세 가지 관점*(놀라움 측정 / KL divergence / MLE)으로 자기 말로 답해 보세요.

TOPIC 4. 체인룰과 자동 미분(Autograd)

4주제 한 줄
우리가 C1에서 *손으로* 한 6단계 역전파를, 컴퓨터는 *임의 깊이의 망*에 어떻게 자동 적용하는가?
우리 수업의 어디와 이어지나
  • C1에서 입1·은1·출1 모델(뉴런 하나로 2층 쌓은 최소 모델)의 모든 미분을 직접 손계산: e_h = e · w₂ · ReLU′(z₁), dw₁ = e_h · x.
  • 은닉이 1층이 아니라 10층, 100층이면 어떻게? 식을 사람이 매번 적을 수 없다 — 컴퓨터가 *그래프로 자동화*한다.
탐구 관점 질문
  • 체인룰의 일반 모양: ∂L/∂w₁ = (∂L/∂ŷ) · (∂ŷ/∂h) · (∂h/∂z₁) · (∂z₁/∂w₁). C1의 6단계가 이 식의 어느 항에 해당하는지 *직접 매핑*해 보세요.
  • "계산 그래프(computation graph)"는 forward 연산을 *노드와 엣지*로 기록한다. backward는 그 그래프를 *역방향*으로 돈다. 이게 왜 *임의 깊이*에서도 작동하는가?
  • PyTorch의 loss.backward() 한 줄이 우리 6단계에 해당한다. 그 한 줄이 *내부적으로 무엇*을 하는가? (그래프 순회 + 미분 합)
  • forward 시 *중간 활성값*을 모두 저장해야 backward가 가능하다. 깊은 망에서 *메모리*가 부족하면? (gradient checkpointing)
  • "역방향 모드 자동 미분" vs "정방향 모드 자동 미분" — 둘은 어떤 상황에서 각자 유리한가?
주요 개념 chain rule computation graph automatic differentiation (autograd) backward / forward mode gradient checkpointing stop_gradient Jacobian
자료 찾기
  • PyTorch 공식 튜토리얼 "A Gentle Introduction to torch.autograd"
  • Andrej Karpathy의 "micrograd" 라이브러리 — 200줄 파이썬으로 autograd 구현
  • "Backpropagation" 위키피디아 — 역사와 일반화된 식
미니 실험 — micrograd 직접 따라 만들기 (도전)
파이썬 200줄 안에 *Value 클래스*(forward 시 그래프 기록 + backward 시 역방향 미분)를 직접 만들어 보세요. C1과 똑같은 모델을 그 위에서 학습시켜 보면, 우리의 6단계 손계산이 자동화되는 모습이 보입니다.
내 정리 — 우리의 C1 6단계가 임의 깊이 망에 *어떻게 일반화*되는지, *체인룰*과 *계산 그래프* 두 단어를 반드시 사용해 자기 말로 한 문단 적어 보세요.

TOPIC 5. 분포 불일치와 일반화 격차

5주제 한 줄
학습 데이터에선 98%인데 *내가 직접 그린 글씨*에선 절반도 못 맞히는 이유는? "외운 것"과 "일반화한 것"은 어떻게 다른가?
우리 수업의 어디와 이어지나
  • C2(MNIST 도전)에서 학습 정확도와 평가 정확도의 차이(일반화 격차)를 봤다.
  • "직접 그린 숫자 테스트"에서 학습 데이터엔 잘 맞으면서도 내 손글씨는 자주 틀리는 현상을 직접 경험했다 — 학습 분포와 실제 사용 분포가 다르다는 뜻.
  • "가르치기"·"증강"으로 약점이 줄어드는 모습까지 봤지만, 이게 *왜 작동*하는지(증강이 분포를 어떻게 넓히는지)는 안 다뤘다.
탐구 관점 질문
  • "외운다(memorization) vs 일반화한다(generalization)"의 차이는 무엇인가? 학습 데이터에선 완벽한데 평가 데이터에선 망가지는 모델은 *어느 쪽*인가?
  • 학습 분포와 실제 사용 분포가 다른 현상(distribution shift): MNIST 학습 데이터와 *내 손글씨*는 어떤 점에서 다른가? (글씨 굵기, 위치, 기울기, 종이 vs 화면…)
  • 데이터 증강(rotation·translation·scale·shear)이 일반화를 돕는 *직관적 이유*는? 모델이 보는 *분포의 폭*을 어떻게 바꾸는가?
  • 모델을 더 깊고 크게 만들면 학습 정확도는 오르지만 일반화 격차도 커지기 쉽다. 이 trade-off를 *bias-variance*로 어떻게 설명하나?
  • "전이 학습(transfer learning)"은 한 분포에서 학습한 모델을 *다른 분포*에 갖다 쓰는 방법이다. 왜 처음부터 새로 학습하는 것보다 자주 더 잘 작동하나?
주요 개념 generalization gap memorization vs generalization distribution shift / covariate shift data augmentation overfitting / underfitting bias-variance trade-off regularization (dropout · weight decay) transfer learning
자료 찾기
  • "Deep learning generalization" 위키피디아 또는 한국어 "딥러닝 일반화"
  • "Data augmentation" — torchvision.transforms 공식 문서의 변환 종류
  • "Distribution shift" 키워드로 covariate shift / dataset shift 글 찾기
  • "Bias-variance tradeoff" — 시각화 자료 (학습 곡선 두 개 그래프)
