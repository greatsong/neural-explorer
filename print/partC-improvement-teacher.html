역전파 한 사이클 손계산 + 사슬규칙 식 유도 + MNIST·증강의 교사용 답안
앱 페이즈 C1~C3 · 학생 활동지 partC-improvement-student-problems.html에 대한 모범 답안과 해설
학생이 손으로 채우는 모든 빈칸·표·서술에 대한 모범 답안과 왜 그렇게 되는지 짧은 해설을 함께 제공합니다. 자기 학습용으로도, 교사 채점용으로도 쓸 수 있습니다.

0. 한 사이클이 무엇인가 [정답·해설]

이 단원의 한 줄 요약
인공 뉴런 2개의 직렬 모델에서 데이터 1쌍(x=2, y=5)을 두고 6단계 — 예측 → 오차 → 출력 쪽 기울기 → 거꾸로 흐른 신호 → 입력 쪽 기울기 → 갱신 — 을 돌리면 손실이 5.61에서 0.37로 줄어듭니다. 이 한 사이클이 모든 깊은 망의 기본 단위.
0-1모델 구조 그리기 — 화살표·라벨 [정답]
학생은 다이어그램 위에 화살표(→)를 그리고 가중치·편향 라벨, 노드 이름을 채웁니다. 모범 답안은 다음과 같습니다.
x —(w₁, b₁)→ Σ → z₁ —(ReLU)→ h —(w₂, b₂)→ Σ → z₂ → ŷ
화살표 라벨: 첫 화살표 = w₁, 두 번째(Σ→ReLU 사이)는 그대로 z₁, 세 번째 = w₂, 마지막 출력은 z₂ = ŷ. 편향 b₁·b₂는 각 Σ 노드 위에 별도 입력선으로 표기.
한 줄 정리 — 이 모델의 식: ŷ = w₂ · ReLU(w₁·x + b₁) + b₂
"깊은 망"이란 이름은 어디서 나왔나? — 곱·합·활성화 한 묶음을 2번 적용해서, 뉴런이 직렬로 2개이기 때문이다.
해설
"깊다(deep)"는 단지 층 수를 말합니다. 1층(perceptron) → 2층 이상이면 "깊은 망". 이 활동지의 모델은 가장 작은 깊은 망(2층)이고, 식의 모양은 GPT의 1750억 파라미터 모델까지 동일합니다.

STEP C1. 한 사이클을 손으로 — 6단계 [정답·해설]

C1-1. ① 예측 — x를 두 뉴런에 통과시키기

C1-1z₁ → h → z₂ → ŷ [정답]
① z₁ = w₁·x + b₁ = 0.5 · 2 + 0.1 = 1.1
② h = ReLU(z₁) = max(0, 1.1) = 1.1
③ z₂ = w₂·h + b₂ = 1.5 · 1.1 + 0.0 = 1.65
④ ŷ = z₂ = 1.65  (회귀라 활성화 없음)
관찰 — z₁이 (양수) → ReLU′(z₁) = 1. 길이 (열려) 있다 → 뉴런 1이 다음 층까지 신호를 전달한다.
왜 이 한 사실이 중요한가 한 줄로: ReLU′(z₁)=1이라 ④-b 단계에서 e_h가 그대로 e_z₁이 되어 뉴런 1의 가중치도 갱신된다.
해설
A1에서 본 인공 뉴런 한 개의 곱·합이 두 번 반복된 모양. z₁이 양수라는 점이 핵심 — 음수였다면 ReLU 문지기에서 막혀 e_z₁=0, 뉴런 1은 학습 불가(죽은 ReLU).

C1-2. ② 오차 — 예측과 정답의 차

C1-2e = ŷ − y [정답]
e = ŷ − y = 1.65 − 5 = −3.35
손실 ½·e² = ½ · (−3.35)² = 5.6113 ≈ 5.61
e의 부호는 (음수). 한 줄 의미: 모델이 정답보다 (작게) 예측 중 → ŷ을 더 키워야 한다.
해설 — 왜 e 한 숫자가 출발점인가
다음에 등장할 dw₂, db₂, e_h, e_z₁, dw₁, db₁ 여섯 숫자가 모두 e에 무엇인가를 곱한 형태입니다. 그래서 e의 부호가 4개 가중치 모두의 갱신 방향(키울까/줄일까)을 결정. 예시 답: "e가 음수면 4개 가중치를 모두 키우는 방향으로 움직인다."
채점 기준: "부호" 또는 "방향"이 들어갔는지 / e가 모든 기울기에 곱해진다는 사실을 짚었는지.

C1-3. ③ 출력 쪽 기울기 — 뉴런 2의 dw₂·db₂

C1-3dw₂ = e·h, db₂ = e [정답]
dw₂ = e · h = (−3.35) · (1.1) = −3.685
db₂ = e · 1 = −3.35
왜 식이 e·h인가 — 예시 답안: A4의 dw = e·x에서 "그 층의 입력"이 x가 아니라 뉴런 1의 출력 h로 바뀐 것이다.
해설
한 층 분의 dw 식은 항상 "그 층의 오차 × 그 층의 입력". 뉴런 2 입장에서 입력은 h. b는 항상 1을 곱한 상수항이라 db = e가 됩니다.
채점 기준: "그 층의 입력" 또는 "h가 뉴런 2의 입력"이라는 표현이 들어갔는지.

C1-4. ④ 거꾸로 흐른 신호 — w₂ 거꾸로 → ReLU 거꾸로 ★ 역전파 핵심

C1-4두 번 거꾸로 통과 — e_h = e·w₂ → e_z₁ = e_h·ReLU′(z₁) [정답]
④-a. 가중치 w₂ 거꾸로 통과 — h의 오차
e_h = e · w₂ = (−3.35) · (1.5) = −5.025  ← h(은닉 활성값)의 오차 ∂L/∂h
④-b. ReLU 문지기 거꾸로 통과 — z₁의 오차
z₁ = (1.1) > 0이므로 ReLU′(z₁) = 1
e_z₁ = e_h · ReLU′(z₁) = (−5.025) · (1) = −5.025  ← z₁(활성화 전 값)의 오차 ∂L/∂z₁
발견! — e_h와 e_z₁이 같은 수치인 이유 [정답]
ReLU′(z₁) = 1이라 e_h에 1을 곱한 e_z₁은 e_h와 같은 숫자(−5.025)가 됩니다. 그래도 식에서 ReLU′ 자리는 사라지지 않아요 — z₁이 살아 있다(문지기가 열렸다)는 사실을 식이 기록하는 것. 두 값이 같은가요? → 예, 같다 (−5.025)
생각해보기 — 만약 z₁이 음수였다면? [정답]
ReLU′(z₁) = (0)이 되어 e_z₁ = e_h · (0) = 0. 그러면 뉴런 1의 가중치는 (움직이지 않는다). 이때 e_h는 (살아 있음) — 가중치 w₂를 거꾸로 통과한 신호 자체는 살아 있지만 ReLU 문지기에서 막혀버린 것. 이런 현상의 이름: 죽은 ReLU (dying ReLU).
관찰 — 지금 |e_h|=5.025는 |e|=3.35보다 (크다). 그 까닭은 곱해진 w₂가 1보다 (큰) 값이기 때문이다 → w₂=1.5가 신호를 1.5배로 증폭시켜 거꾸로 보냈다.
해설 — 이 카드가 왜 핵심인가
뉴런 1은 자기 정답이 없어 직접 e를 계산할 수 없습니다. 그래서 출력의 e가 두 번 거꾸로 흘러 뉴런 1까지 책임을 운반: ① 가중치 w₂를 거꾸로 통과해 h의 오차 e_h, ② ReLU 문지기를 거꾸로 통과해 z₁의 오차 e_z₁. 이 거꾸로 흐름이 곧 역전파(backpropagation). 순전파에서 ReLU 문지기를 만났던 것처럼, 역전파에서도 같은 위치에서 같은 문지기를 만납니다 — 이번엔 z₁>0이라 둘 다 통과하지만, 음수 케이스에선 ②에서 막힙니다.
채점 기준: ④-a 식·결과(2점), ④-b ReLU′ 값+e_z₁ 결과(2점), 두 값이 같은 이유(2점), 음수 케이스 답(2점). 총 8점.

C1-5. ⑤ 입력 쪽 기울기 — 뉴런 1의 dw₁·db₁

C1-5dw₁ = e_z₁·x, db₁ = e_z₁ [정답]
dw₁ = e_z₁ · x = (−5.025) · (2) = −10.05
db₁ = e_z₁ · 1 = −5.025
관찰 — 이 패턴이 (모든) 층에 똑같이 적용된다. 자기 말로 한 줄: 층마다 "그 층의 오차 × 그 층의 입력"이라는 같은 모양으로 dw가 결정된다.
해설
뉴런 1과 뉴런 2의 식이 둘 다 d_가중치 = (그 층의 오차) × (그 층의 입력) 모양. 뉴런 2는 (e, h), 뉴런 1은 (e_z₁, x). 단지 "그 층의 오차"가 출력층은 e, 그 앞 층은 e_z₁로 바뀔 뿐.
채점 기준: "모든 층" 또는 "같은 모양"이라는 일반화 표현이 들어갔는지.

C1-6. ⑥ 갱신 — 4개 가중치를 동시에

C1-6w ← w − η·dw  (η = 0.05) [정답]
가중치현재값기울기η · 기울기새 값
w₂1.5−3.685−0.184251.684
b₂0.0−3.35−0.16750.168
w₁0.5−10.05−0.50251.003
b₁0.1−5.025−0.251250.351
관찰 — 4개 dw·db가 모두 (음수)였으므로, 빼기를 거쳐 가중치는 모두 (커졌다). 이는 e가 (음수)였기 때문에 모델이 ŷ을 (키우는) 방향으로 움직인 것이다.
해설
"w ← w − η·dw"의 빼기가 핵심. dw가 음수면 −(음수)=+가 되어 w가 커지고, 양수면 줄어듭니다. e가 한 부호로 통일되어 있어 4개 가중치가 같은 방향으로 움직였어요. 학습률 η=0.05가 한 걸음의 크기를 결정.

C1-7. 갱신 후 — 손실이 정말 줄었는가? [정답]

C1-7새 가중치로 같은 x = 2를 다시 통과 [정답]
갱신 전 손실 ½·e² = 5.61. 새 가중치(w₁=1.003, b₁=0.351, w₂=1.684, b₂=0.168)로 다시 통과:
새 z₁ = 1.003 · 2 + 0.351 = 2.357
새 h = ReLU(2.357) = 2.357
새 z₂ = 1.684 · 2.357 + 0.168 = 4.136
새 ŷ = 4.136,   새 e = ŷ − 5 = −0.864
새 손실 ½·e² = ½ · 0.7465 ≈ 0.373
새 손실은 갱신 전 손실보다 (작다). 자기 말로 한 줄: 한 사이클(한 step)을 한 번만 돌렸을 뿐인데 손실이 5.61 → 0.37로 약 1/15로 줄었다 — 이 사이클이 학습 그 자체이다.
해설
손실이 정말로 줄었다는 사실이 학습이 작동한다는 결정적 증거. 정답 ŷ=5에서 1.65 → 4.14로 가까워졌고, 사이클을 더 돌리면 5에 점점 다가갑니다.
채점 기준: "손실이 줄었다" + 그 의미 한 줄(학습이 작동한다 / ŷ이 정답에 가까워졌다 등)이 들어갔는지.

C1-8. 한 사이클 흐름 띠 [정답]

C1-8한 step의 6단계를 한 줄로 [정답]
① 예측 (ŷ) → ② 오차 (e) → ③ 출력 쪽 기울기 (dw₂, db₂) → ④ 거꾸로 흐른 신호 (e_h, e_z₁) → ⑤ 입력 쪽 기울기 (dw₁, db₁) → ⑥ 갱신 (w ← w − η·dw)
"역전파의 핵심"은 ④번 단계. 까닭: 출력의 오차 e가 두 번 거꾸로(w₂를 거꾸로, ReLU 문지기를 거꾸로) 통과해 뉴런 1까지 책임을 가져오는 단계이기 때문 — 이 거꾸로 흐름이 곧 backpropagation의 정의.
자기 말로 두 줄 — 예시 답안:
예시 답안
출력층의 오차 e가 가중치 w₂를 거꾸로 통과해 h의 오차 e_h가 되고, 다시 ReLU 문지기를 거꾸로 통과해 z₁의 오차 e_z₁이 된다. 이번엔 z₁>0이라 ReLU′=1이어서 e_h와 e_z₁이 같은 수치가 되지만, 통과 자체는 식에 명시된다. 이렇게 두 번 거꾸로 흘러 운반된 신호가 뉴런 1의 가중치를 갱신하는 출발점이 된다.
채점 기준 (4개 단어 사용)
  • "w₂"가 들어갔는가
  • "거꾸로"가 들어갔는가
  • "ReLU′"가 들어갔는가
  • "문지기"가 들어갔는가
4개 중 3개 이상이면 만점, 2개면 부분 점수.

2 사이클 후 ŷ ≈ 4.96 계산: 새 z₁ = 1.148·2 + 0.424 = 2.720, h = 2.720, z₂ = 1.786·2.720 + 0.211 ≈ 5.069. 따라서 ŷ ≈ 5.07, e ≈ +0.07, 손실 ≈ 0.0024 (학생 자릿수에 따라 ≈0.001~0.003). 이미 정답 5를 살짝 지나칠 수 있어요(이래서 학습률 조절이 필요). 이 시점부터는 보통 학습률을 줄이거나 step 수를 늘려 미세 조정합니다.
채점 기준: 손실이 계속 줄어든다는 사실 + 감소 폭이 작아진다는 관찰 + 그 까닭을 e의 감소와 연결했는지.

STEP C2. 식 유도 — 사슬규칙으로 한 줄씩 [정답·해설]

이 단원의 한 줄 요약
A4의 단일 뉴런 dw = e·x를 만든 사슬규칙이 두 층에 자연스럽게 확장됩니다. e_h = e·w₂, e_z₁ = e_h·ReLU′(z₁), 그리고 모든 dw·db에서 *그 층의 입력*이 곱해진다는 공통 패턴이 한 줄로 정리됩니다. 학생이 식을 외우는 게 아니라 스스로 다시 끌어낼 수 있는 수준이 목표.

C2-1. 합성 함수 옮겨 적기 [정답]

C2-1x → ŷ → L의 식 4줄 [정답]
L = ½(ŷ − y)²
ŷ = z₂ = w₂ · h + b₂
h = ReLU(z₁)
z₁ = w₁ · x + b₁
e의 정의: ½ · 2 · (ŷ−y) = ŷ − y = e. ½과 2가 약분되어 e가 깔끔하게 나오는 것이 ½을 도입한 이유.
해설
이 4줄이 모든 미분의 출발점. 합성 함수가 명확히 적혀 있으면 사슬규칙은 *기계적*으로 따라온다. 학생이 "왜 ½인가?"를 물으면 — A2에서 미분 결과가 깔끔해지도록 도입한 약분 장치라는 점을 다시 짚어줄 것.
채점 기준: 4식 모두 정확(각 1점, 총 4점), e = ŷ−y(2점). 총 6점.

C2-2. ★ e_h 유도 [정답]

C2-2e_h = ∂L/∂h [정답]
∂L/∂ŷ = e  (C2-1에서)
ŷ = w₂h + b₂에서 ∂ŷ/∂h = w₂
e_h = (e) · (w₂) = e · w₂
관찰 한 줄 정답 예시: "순전파에서 h가 w₂배로 증폭돼 ŷ에 도달했으므로, 역전파에서도 ŷ의 변화가 같은 w₂배로 h에 되돌아간다. 같은 가중치 w₂가 두 방향으로 작용한다."
검산: e_h = (−3.35) · (1.50) = −5.025 ≈ −5.02. C1-4의 값과 같음 ✓
해설 — 이 한 줄이 역전파의 핵심
"w₂를 거꾸로 통과"라는 비유의 정확한 의미가 ∂ŷ/∂h = w₂에서 나온다. 학생이 이 한 줄을 *외우지 않고 다시 끌어낼 수 있다면* 다른 활성함수·다른 구조에서도 그래디언트를 직접 적을 수 있다.
채점 기준: 빈칸 4개 정답(각 1점), 관찰 한 줄(2점), 검산(1점). 총 7점. 핵심 문제이므로 가중치 높음.

C2-3. ★ e_z₁ 유도 [정답]

C2-3e_z₁ = ∂L/∂z₁ [정답]
∂h/∂z₁ = ReLU′(z₁) = 1  (z₁ = 1.10 > 0이므로)
e_z₁ = e_h · (1) = e_h = −5.025
e_h와 e_z₁의 관계 — 정답 예시: "이번엔 ReLU′ = 1이라 두 값이 같지만, ReLU 통과라는 *단계 자체*는 식에 명시된다. z₁ ≤ 0이면 ReLU′ = 0이 되어 신호가 끊기고 뉴런 1 가중치가 갱신 안 된다(Dying ReLU)."
해설
이 단계가 *활성함수가 역전파에서 등장하는 자리*. 학생이 ReLU′를 빠뜨리고 e_h를 곧장 dw₁·db₁에 쓰면 *Dying ReLU 케이스를 설명할 수 없다*. 이 점을 강조.
채점 기준: 빈칸 3개(각 1점), 관계 한 줄(2점, "통과 단계 명시" + "음수일 때 끊김"). 총 5점.

C2-4. dw·db 일반 패턴 [정답]

C2-4모든 d_가중치 = (그 층의 오차) × (그 층의 입력) [정답]
∂L/∂w₂ = e · h = −3.685  (C1-3과 일치 ✓)
∂L/∂b₂ = e · 1 = −3.35
∂L/∂w₁ = e_z₁ · x = −10.05  (C1-5와 일치 ✓)
∂L/∂b₁ = e_z₁ · 1 = −5.025
한 패턴 정답 예시: "사슬규칙으로 풀어보면 모든 층의 d_가중치가 (그 층의 오차) × (그 층의 입력) 한 패턴으로 정리된다. 출력층에선 e와 h, 입력층에선 e_z₁과 x. 거꾸로 흐른 신호가 그 층까지의 *오차*로 쓰인다는 점만 다르다."
해설
A4의 단일 뉴런 식이 다층에 그대로 확장됨을 학생이 *식으로* 확인하는 자리. C1에서 손으로 푼 값(−3.685, −10.05 등)이 사슬규칙으로도 같게 나온다는 *일치 경험*이 핵심.
채점 기준: 4식 정답(각 1점, 총 4점), 한 패턴 서술(3점, "사슬규칙" + "그 층의 입력" + "거꾸로" 단어 포함). 총 7점.

C2-5. 한 문장 요약 [정답]

C2-5역전파 = 합성 함수의 미분 [정답]
예시 답안
역전파는 합성 함수의 미분이다 — 손실 L까지의 합성 식 (L → ŷ → h → z₁ → x)을 사슬규칙으로 풀면, 출력 오차 e가 *가중치 거꾸로* (× w₂)와 *ReLU 문지기* (× ReLU′(z₁)) 두 단계를 거꾸로 통과하며 입력층까지 흘러간다. 모든 층의 d_가중치는 (그 층의 오차) × (그 층의 입력) 한 패턴으로 같다.
채점 기준 (4개 단어 사용)
  • "합성 함수"가 들어갔는가
  • "사슬규칙"이 들어갔는가
  • "가중치 거꾸로" 또는 "w₂ 거꾸로"가 들어갔는가
  • "ReLU 문지기" 또는 "ReLU′"가 들어갔는가
4개 모두 들어가면 8점, 3개면 6점, 2개면 3점.

STEP C3. MNIST와 데이터 증강 [정답·해설]

C3-1. B5와 MNIST를 나란히 [정답]

채점 기준: 입력 크기 + 라벨 수 둘 다 짚었는지.

C3-2. "내가 그린 숫자가 안 맞는 까닭" [정답]

C3-2학습 데이터와 내 그림은 무엇이 다른가 [정답]
① 학습 데이터는 (중앙 정렬된) 28×28인데, 내 그림은 보통 (자유 위치) 280×280이라 크기·해상도가 다르다.
② 단순 다층 퍼셉트론은 픽셀의 (위치)에 민감해서, 한쪽으로 치우쳐 그린 글씨를 다른 숫자로 본다.
③ 모델의 (확신도(softmax 최대값))가 50%대로 낮으면, 후보끼리 박빙이라 작은 차이로도 답이 바뀐다.
한 문장 합치기 — 예시 답안: 내 그림이 학습 분포(중앙 정렬·일정한 굵기·28×28)와 다른 위치·해상도·기울기로 들어가는데, 모델이 픽셀 위치에 민감하고 확신도까지 낮아서 자신 있게 틀린다.
해설
①은 "정렬·해상도", ②는 "위치 민감성", ③은 "확신도"가 핵심 키워드. ②와 C3-5의 답이 직접 연결됩니다.
채점 기준: ①에 "정렬"·"중앙"·"28×28" 중 하나, ②에 "위치"·"공간", ③에 "확신도"·"softmax"·"확률"이 들어갔는지.

C3-3. 데이터 증강 4가지 — 효과 짝짓기 [정답]

C3-34가지 변형의 짝 [정답]
변형예시 범위짝효과 — 모범 답안

스케일0.8~1.2배㉢크고 작은 글씨 모두 같은 숫자로 인식

해설
회전=각도, 이동=위치, 스케일=크기, 전단=비스듬함 — 변환의 이름과 효과의 키워드가 1:1로 대응합니다. 학생이 효과 칸을 자기 말로 적었는지가 채점 포인트.
채점 기준: 짝 4개 모두 정답이면 만점, 효과 칸은 키워드(기울기/위치/크기/비스듬함)가 보이면 정답.

C3-4. 1장 → N장 → 재학습 [정답]

C3-4틀린 그림 1장으로 시작 [정답]
틀린 그림 1장은 본래 학습 데이터에 (없던 / 부족했던) 분포의 그림이었다.
→ 한 장만 그대로 추가하면 모델은 그 한 장을 (외워(과적합해)) 버리기 쉽다.
→ 그래서 그 한 장을 N장으로 (증강(변형 복제))해 분포의 폭을 넓힌다.
→ 다시 학습하면, 그 분포의 그림에서도 (일반화(올바르게 분류))할 수 있게 된다.
N을 너무 작게 → (여전히 외움(과적합)), 너무 크게 → (학습이 느려짐 / 다른 데이터를 압도) 위험.
★ 다시 학습할 때, 변하는 것은 (학습 데이터의 분포 폭(입력의 다양성))이고, 변하지 않는 것은 (한 사이클의 6단계 — 예측·오차·기울기·갱신 식)이다.
해설
이 카드의 핵심 일반화: "사이클 식은 그대로, 데이터만 바뀐다." C1과 C2를 잇는 다리.
채점 기준: 변하는 것에 "데이터/분포", 변하지 않는 것에 "사이클/식/6단계"가 들어갔는지.

C3-5. 같은 4를 어디에 그릴까 [정답]

C3-5위치가 다르면 픽셀 입장에선 다른 입력 [정답]
예측 — 더 잘 맞힐 쪽: (나) 가운데에 크게.
이유 — 예시 답안: 학습 분포(MNIST)는 숫자가 중앙에 정렬되어 있어, 단순 MLP가 "4의 픽셀은 가운데 어느 자리"라는 식으로 픽셀 위치를 외운다. (가)처럼 왼쪽 위에 작게 그리면 픽셀 위치가 학습 분포와 어긋나 다른 숫자로 본다.
처치 한 단어 예측: 이동 증강 (translation augmentation)
해설
C3-2의 ②와 직결되는 카드. "픽셀 위치"와 "학습 분포" 두 단어가 핵심. 처치는 이동 증강 또는 단순히 "증강".
채점 기준: "픽셀 위치"·"학습 분포" 두 단어 사용 / 처치 답에 "이동" 또는 "증강"이 들어갔는지.

C3-6. "외운 것 vs 일반화한 것" [정답]

C3-6학습 정확도 98% vs 내 그림 50% [정답]
예시 답안 (5~6줄)
학습 정확도는 학습 분포 안의 데이터에 대한 점수이고, 내 그림은 학습 분포와 다른 사용 분포에 속한다. 두 분포가 어긋나면 모델은 "외운 패턴"만으로는 새 입력을 풀지 못한다. 증강은 학습 분포의 폭을 사용 분포 쪽으로 넓혀, 외운 것이 아니라 같은 숫자의 여러 변형까지 포함한 일반화로 바꾼다.
"외운 것"과 "일반화한 것"의 차이 한 줄: 외운 것은 학습 데이터에서만 작동하지만, 일반화한 것은 학습 분포 밖의 변형(위치·크기·기울기)에도 같은 답을 낸다.
채점 기준 (3개 단어 사용)
  • "학습 분포"가 들어갔는가
  • "사용 분포"가 들어갔는가
  • "증강"이 들어갔는가
세 단어 모두 들어가면 만점, 두 개면 부분 점수.

★★2. 같은 7을 두 가지로 그려보기 [정답]

예시 답안 — 눈으로 본 차이: (가)는 28×28 가운데에 똑바른 7이 들어가 학습 데이터와 모양이 거의 같지만, (나)는 한쪽 구석에 비스듬히 7이 그려져 픽셀 단위로는 거의 다른 그림처럼 보인다.

해설
실제로 7을 두 가지로 그려본 학생이라면, 같은 숫자라는 사실이 사람 눈에는 분명해도 픽셀 벡터로는 코사인 유사도가 크게 떨어집니다. 이 체험이 "분포 불일치"의 직관.
채점 기준: 차이를 한 줄로 적었는지 + 측면 1가지를 4개 보기 안에서 골랐는지.

★★3. 분포의 폭이 넓어진다 [정답]

★★3한 장 vs 한 장 + 증강 N장 [정답]
학생은 오른쪽 박스에 점 4~6개를 더 흩뿌려 그립니다. 점들이 가운데 점 주변으로 흩어지면 정답.
두 박스의 가장 큰 차이: 분포의 (폭(너비))이 (넓어)졌다.
일반화에 필요한 것: (분포의 폭) — 단순한 양이 아니라 폭이 핵심.
예시 답안: 같은 그림 1000장을 복제해 양만 늘리면 모델은 그 한 점을 더 강하게 외울 뿐이다. 변형으로 점들을 흩어 분포의 폭을 넓혀야 모델이 "그 영역 안의 어느 점이든 같은 답"이라고 일반화할 수 있다.
해설
"양 vs 폭"의 구분이 이 카드의 핵심. 같은 한 점을 N번 복제 = 양 증가, 변형으로 흩뿌리기 = 폭 증가.
채점 기준: "폭" 또는 "넓이/다양성"이라는 단어가 들어갔는지 / 단순한 양과 구별했는지.

★★5. 한 사이클과 분포를 잇는 한 문장 [정답]

딥러닝의 학습은 한 한 사이클의 반복이다. 한 사이클은 예측 → 오차 → 출력 쪽 기울기 → 거꾸로 흐른 신호 → 입력 쪽 기울기 → 갱신의 6단계로, 출력의 오차 e가 가중치 w₂를 거꾸로 통과해 뉴런 1까지 책임을 운반하는 절차를 역전파라 부른다. 손글씨(MNIST)에서도 같은 사이클이 그대로 작동해 98%까지 정확도가 오를 수 있다. 하지만 직접 마우스로 그린 숫자에서는 자주 틀리는데, 이는 학습 데이터(중앙 정렬·일정한 굵기)와 내 그림(자유 위치·자유 굵기) 사이의 분포 불일치 때문이다. 이 격차를 메우려면 틀린 그림을 회전·이동·스케일·전단으로 N장 변형하는 데이터 증강으로 학습 분포의 폭을 넓히면 된다. 사이클 식은 그대로 두고 데이터만 다양해지면, 모델은 외운 패턴이 아니라 일반화된 패턴을 배우게 된다.

  • 밑줄 표기 5개
  • 문장 수 5~7
  • "한 사이클이 무엇인가" + "MNIST에서 왜 틀리는가" + "어떻게 더 잘 맞히게 하는가" 세 질문 모두에 답했는지
