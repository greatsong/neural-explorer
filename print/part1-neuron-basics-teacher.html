1부 — 뉴런의 기초

앱 페이즈 1~5 · 학생용과 동일 구조 + 정답·해설·관찰 평가 기준
핵심 메시지 학생이 1부를 끝낼 때 "학습 = 손실을 줄이는 방향으로 w·b를 조금씩 미는 일"을 자기 말로 말할 수 있어야 한다. 수식 자체보다 흐름에 집중시킬 것.

STEP 2. 순전파 퀴즈 (Phase 2)

2-1문제 ① 정답
정답z = 1·2 + 2·3 + (−1) = 7  /  y = ReLU(7) = 7
2-2문제 ② 정답
정답z = (−1)·4 + 3·1 + 2 = 1  /  y = ReLU(1) = 1
2-3문제 ③ 정답
정답z = (−2)·5 + (−1)·2 + 3 = −9  /  y = ReLU(−9) = 0   → "반응하지 않았다"
자주 틀림음수 z를 그대로 답으로 적는 경우. ReLU 단계에서 0으로 잘림을 한 번 더 강조.
수업 팁"이 뉴런은 이번 입력에 침묵했다"라는 표현으로 정리하면, 뒤에 나올 희소 활성 개념의 토대가 된다.

3-2관찰 — 모범 답안
모양아래로 볼록한 포물선(U자형), 정점이 정답(5) 위치에 있고 손실 = 0.
대칭성예측 −1 → 오차 −6 → 손실 36. 예측 11 → 오차 +6 → 손실 36. 같다. 이유: 제곱은 부호를 없앤다.
제곱 3가지 이유① 부호 상쇄 방지 ② 큰 오차에 큰 페널티 ③ 매끄러운 곡선(미분 가능) → 다음 STEP의 자동 학습에서 컴퓨터가 "어느 쪽이 내리막인지"를 정확히 계산할 수 있는 기반.
수업 팁"부호가 사라지면 방향 정보도 사라지지 않을까?" 라는 질문을 학생에게 던질 것 — 제곱 후에도 곡선의 기울기가 방향을 되살리고, 그것이 STEP 5에서 자동으로 계산된다는 복선이 된다.

STEP 4. 수동 학습 (Phase 4)

앱 운영Phase 4가 데이터 y = 2x + 0.7로 바뀌었음을 칠판에 먼저 적어둘 것. 정답 b가 0.5의 배수가 아니라 큰칸 0.5로는 절대 도달 불가한 함정이 의도적으로 들어가 있다.

4-2슬라이더 한 칸 크기 — 모범 답안
한 칸 크기관찰
0.5 (큰칸)b가 0, 0.5, 1.0, 1.5만 가능 → 0.7에 도달 불가. 0.5와 1.0을 오가는 진동.
0.1 (보통)b를 0.7로 정확히 맞춰 정답 도달. 손실 = 0.
0.01 (미세)정답 도달은 가능하지만 같은 거리를 가는 데 칸 수가 많아 답답함.
큰칸 0.5로 못 가는 이유한 칸이 0.5의 배수만 만들 수 있는데 정답 b = 0.7은 0.5의 배수가 아니다. 보폭 자체가 정답을 지나치게(overshoot) 만든다.
전이 메시지슬라이더 한 칸 크기 = STEP 5의 학습률 η(보폭). "너무 크면 정답을 지나친다"는 감각을 학생이 손으로 먼저 체험하게 하는 의도된 함정.
4-3학습 정의 — 모범 답안
예시"학습이란 총 손실이 가장 작아지는 w·b를 찾는 일이다." (다른 표현 허용: "데이터에 가장 잘 맞는 직선의 기울기와 절편을 정하는 일".)

STEP 5. 자동으로 학습하기 (Phase 5)

앱 운영Phase 5는 STEP 4의 슬라이더를 컴퓨터가 대신 움직이는 페이지. 데이터는 (1,3)~(5,11)로 다시 정수, 모델에 ReLU 활성화가 들어감(데이터가 모두 양수+정답 근처라 z>0이라 ReLU 통과는 항등이지만, "신경망의 일반 모습"을 보여주는 의도). 한 step = 1 에포크라는 용어를 칠판에 먼저 고정.
5-1한 점 변화량 — 정답
계산 z = 1·3 + 0 = 3
ŷ = ReLU(3) = 3  (z > 0이라 그대로 통과)
e = 3 − 7 = −4
w 변화량 = e · x = (−4)·3 = −12
b 변화량 = e = −4
업데이트: w ← 1 − 0.1·(−12) = 2.2,   b ← 0 − 0.1·(−4) = 0.4
→ 정답 (2, 1)에 가까워졌다.
자주 틀림① 부호 약속(e = ŷ − y) 헷갈려 부호 반대로. ② 마이너스 학습률 부분을 빼먹어 발산 방향으로 업데이트. → "오차 = 예측 − 정답, 업데이트는 −η·변화량"을 칠판 상단에 고정.
수업 팁"한 점만으로 갱신"은 단순화된 시나리오. 실제 앱은 다섯 점 평균으로 갱신한다는 점을 STEP 5-2 진행 직전에 짚을 것.

5-3학습률 비교 — 정답 (앱 슬라이더 범위 0.001 ~ 0.1)
η20단계 후 손실현상
0.001≈ 14~20매우 느림. 20단계로는 정답 근처에 못 감.
0.01≈ 1~2느린 수렴. 손실이 부드럽게 감소하지만 시간이 걸림.
0.05≈ 0.01 부근적절. 약 10~15단계 안에 거의 수렴.
0.1≈ 0.005 이하빠름. 1~2단계의 큰 점프 후 미세 진동하며 수렴.
한 줄 정리η가 작으면 답답하고, 너무 크면 정답을 지나쳐 못 멈춘다. → STEP 4의 큰칸 0.5가 b=0.7에 못 닿던 일과 정확히 같은 현상.
앱 제약앱의 학습률 최대값이 0.1로 제한돼 있어 "발산"을 직접 시연할 수는 없다. 발산을 보여주려면 칠판에서 η = 1.5로 한 step을 손계산해 손실이 폭발적으로 커지는 모습을 보여줄 것.
5-4왜 빼기인가 — 모범 답안
정답변화량(dw, db)은 손실이 가장 빠르게 증가하는 방향을 가리키는 나침반이다. 우리는 손실의 감소를 원하니 그 반대 방향으로 가야 하므로 부호를 뒤집기 위해 빼기를 쓴다.

A 도달 세 부품의 역할을 자기 말로 구분 설명. 음수 가중치/편향의 의미를 예시로 설명. 5문항 이상 모두 정확. 음수 부호·ReLU 처리 정확. 5용어를 모두 자연스럽게 사용해 6~7문장으로 흐름을 설명.
B 진행 세 부품을 식별하지만 음수의 의미가 모호. 1~2문제 부호 실수. 다시 짚으면 스스로 수정. 일부 용어를 빠뜨리거나 순서가 어색.
C 보충 부품 이름은 알지만 식과 연결 못 함. ReLU·부호 처리에서 반복 오류. "손실"·"기울기"의 역할을 설명하지 못함.
전이 포인트1부에서 학습이 수치 회귀(연속 값 맞히기)로 끝났음을 짚고, 2부에서는 같은 도구로 분류(합·불, 0·1)를 다룬다는 점을 예고할 것. 시그모이드 도입의 자연스러운 연결고리가 된다.
