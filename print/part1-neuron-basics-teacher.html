1부 — 뉴런의 기초

앱 페이즈 1~5 · 학생용과 동일 구조 + 정답·해설·관찰 평가 기준
핵심 메시지 학생이 1부를 끝낼 때 "학습 = 손실을 줄이는 방향으로 w·b를 조금씩 미는 일"을 자기 말로 말할 수 있어야 한다. 수식 자체보다 흐름에 집중시킬 것.

STEP 2. 순전파 퀴즈 (Phase 2)

앱 운영Phase 2가 3탭(① 단일 뉴런 → ② 직렬 뉴런 → ③ 3×2×1 작은 신경망)으로 구성. 활동지의 STEP 2도 같은 흐름. 빈칸(?)을 맞히면 다이어그램에 값이 나타나는 점진적 공개 방식이라, 학생이 한 단계 정답을 확인한 뒤 다음 단계로 넘어가게 운영할 것. 핵심 메시지는 "한 뉴런이든 작은 신경망이든 곱셈 → 합산 → ReLU의 절차는 똑같다".

정답 z₁ = 2·3 + (−1) = 5
h = ReLU(5) = 5

자주 틀림z₁이 음수일 때 h를 그대로 음수로 적는 경우(이번 문제는 z₁=5라 발생 안 하지만, 학생이 일반화할 수 있도록 ReLU의 "음수면 0" 규칙을 한 번 더 짚어줄 것).

수업 팁가중치 0인 연결(h₁ ← x₃, h₂ ← x₁)은 앱에서 흐린 점선으로 표시 — "기여 없음"을 시각적으로 보여주는 장치. 학생이 "왜 어떤 선은 흐려요?"라고 물으면 좋은 신호. 이 관찰이 페이즈 11(신경망 설계)에서 "모든 입력이 모든 뉴런에 연결되지만 가중치 크기로 영향력이 다르다"라는 포인트로 자연스럽게 연결됨.
자주 틀림은닉층 두 뉴런을 한꺼번에 계산하려다 항을 빠뜨리는 경우. 한 뉴런씩 따로 식을 쓰게 시범 보일 것.

STEP 3. 손실함수와 경사하강법의 이해 (Phase 3)

앱 운영Phase 3가 5탭(① 데이터 → ② 오차 → ③ 오차² → ④ 손실(MSE) → ⑤ 경사하강법)으로 재구성됐다. 같은 다섯 점에 가장 잘 맞는 직선을 찾는 단일 흐름. 활동지의 STEP 3은 손계산을 깔끔하게 하기 위해 손실식을 L = (w − 5)²로 단순화함을 학생에게 먼저 안내할 것 — 앱은 다섯 점에 직선을 맞추는 정식 MSE를, 활동지는 가장 단순한 포물선을 다루지만 "포물선 모양 + 기울기 반대 방향으로 이동"이라는 핵심 발상은 같다.
앱의 모델 개념Phase 3 데이터 탭에 "📌 모델이 뭐예요?" 박스가 새로 들어왔다 — 모델 구조 + 매개변수 + 학습의 3단 분리. 활동지 STEP 3 도입에도 같은 박스가 들어 있으니 학생이 활동지 박스를 먼저 읽고 앱에서 같은 박스를 발견하게 운영하면 인지 부담이 적다.
3-1제곱 3가지 이유 — 정답
정답① +3과 (−3)을 그대로 더하면 (0)이 되어 "오차 없음"처럼 보임. 제곱하면 모두 (양수)가 됨.

3-8학습 종료 신호 — 정답
정답w = 5에서 기울기 = 0이므로 갱신 식 w ← w − η · 기울기에 넣으면 새 w = 5 − η · 0 = 5 → η가 어떤 값이든 더 이상 변하지 않는다. 골짜기 바닥에 도달했다는 뜻.

STEP 5. 오차 역전파의 이해 (Phase 5)

앱 운영Phase 5가 5개 탭으로 재구성됐다(① 직관 → ② 수식 이해 → ③ 수식 유도 → ④ 전체 종합 → ⑤ 실생활 적용). 탭 ②에서 5-1·5-2를, 탭 ④에서 5-3·5-4를, 탭 ③에서 5-5를, 탭 ⑤에서 5-6을 푸는 흐름으로 진행. 한 차시 안에 5탭을 모두 도는 게 부담스러우면 ①·②·④만 진행하고 ③·⑤를 과제·심화로 미뤄도 무방.
수업 팁탭 이름이 "수식 유도"·"실생활 적용"으로 무거워 보이지만, 학생이 직접 식을 세우는 건 아니다. 앱이 11단계로 풀어주는 걸 따라 읽고 핵심 한 줄을 옮겨 적는 활동, 두 모델의 결과를 비교 관찰하는 활동이다.
학생용 5-A~5-E 절 매칭   5-1(결과 표 패턴 발견) · 5-2~5-5(한 step 손계산) · 5-6~5-7(책임 분담 직관) · 5-8~5-9(학습률·빼기) · 5-10~5-11(실생활 적용). 활동지가 앱 없이 자력으로 풀 수 있도록 결과 표·시나리오를 모두 활동지 안에 직접 제공.

5-1 표와의 차이 짚기5-1 표의 step 1은 시작점이 (w=0, b=0)이라 결과가 (w=2.50, b=0.70). 5-5는 시작점이 (w=1, b=0)이라 (w=2.4, b=0.4). 같은 절차가 시작점에 따라 다른 결과를 낸다는 점을 짚어줄 것.
자주 틀림업데이트 식의 빼기 부호를 빼먹어 1+0.1·(−14) = −0.4 같은 발산 방향으로 답하는 경우.
5-6·5-7책임 분담 — 모범 답안
5-6 자기 말로 풀이 — 예시"곱셈에서 곱해지는 수가 클수록 w가 흔들렸을 때 결과 변화가 크니까, x가 큰 점이 w를 더 강하게 끌어당긴다." 또는 "x=5인 점에서 w를 0.1 흔들면 출력이 0.5 흔들리지만, x=1에서는 0.1만 흔들리니, 그만큼 책임이 다르다."
5-7 5-4 표로 확인 — 정답같은 오차 −1이라도 x=1에서 e×x = −1이지만 x=5에서 e×x = −5. x가 큰 점이 |e×x|가 더 크다. 점 (5, 11)이 점 (1, 3)보다 더 큼.
자주 틀림"미분이라서 x가 곱해진다"고만 답하는 경우. 의미(책임 분담)를 자기 말로 설명할 수 있어야 만점.
5-8학습률 비교 — 정답
정답 η가 너무 작으면 (느려서 / 답답해서) 못 간다.
η가 적절하면 빠르게 (0 / 정답)에 수렴한다.
STEP 4의 한 칸 크기와 같은 점: 한 발짝의 크기가 너무 크면 정답을 지나치고 너무 작으면 답답하다는 동일한 트레이드오프.
η가 너무 크면 (예: 1.5) — 모범 답안한 step에서 정답을 크게 지나쳐 반대편으로 튕겨 나감. 다시 돌아오려 해도 더 멀리 튕겨 손실이 매번 커지는 발산 발생.
Dying ReLU 시연Phase 5 컨트롤 패널에 "⚠ 뉴런 사망 (Dying ReLU)" 경고 UI가 있다. 학습률을 0.3 이상으로 올리고 20단계 반복을 누르면 한 step에서 w·b가 크게 음수로 튀어 모든 점에서 z < 0 → ReLU(z)=0 → 기울기 0 → 학습 영구 정지. 경고가 뜨면 초기화 버튼이 강조 표시로 바뀐다. "발산만 위험한 게 아니라 죽는 것도 위험"이라는 점을 한 번 시연으로 보여주면 학습률 튜닝의 의미가 강해진다.
5-9왜 빼기인가 — 정답
정답변화량(dw, db)은 손실이 가장 빠르게 증가하는 방향을 가리키는 나침반. 손실의 감소를 원하니 그 반대 방향으로 가야 하므로 빼기를 쓴다. (STEP 3-7에서 본 식과 같은 발상이며, 학습률 η가 추가됐을 뿐.)
5-10두 모델 비교 — 예시 답안
정답모델 B가 더 잘 맞는다. 이유: 직선만으로는 표현할 수 없는 곡선 패턴을 은닉층이 잡아낼 수 있기 때문(표현력이 더 큼).
"B가 무조건 좋다"고 결론 못 짓는 이유 — 모범 답안모델이 커질수록 학습 데이터를 너무 잘 외워버리는 과적합(overfitting) 위험이 커진다. 새 데이터에서는 오히려 손실이 클 수 있다.
자주 틀림"B가 무조건 좋은 모델"로 단순화. "표현력이 클수록 과적합 위험도 커진다"는 단서를 한 마디 보태두면 3·4부의 일반화 논의가 미리 자연스러워진다.
5-11교실 데이터 사례 — 예시 답안
예시① x = 자습 시간 / y = 시험 점수 ② x = 키 / y = 몸무게 ③ x = 이번 주 출석일 / y = 다음 주 자습 시간. 한 변수–한 정답 구조이면 OK.
전이이 절은 "방금 배운 절차가 다른 데이터에서도 그대로 통한다"는 점을 학생이 자기 말로 정리하게 하는 마무리. 정확한 손실 수치보다 "단일 뉴런으로 표현 못 하는 패턴 → 더 큰 신경망의 동기"로 닫으면 STEP 6·2부로 자연스럽게 이어진다.
