6부 후반 — 시퀀스에서 GPT까지

앱 페이즈 19~22 · 학생용과 동일 구조 + 정답·해설
핵심 메시지 "어텐션은 단어들끼리 서로 점수 매겨 가중합하는 일이고, 트랜스포머 = 어텐션 + FFN 블록을 N층 쌓은 것이며, GPT = 그 출력에서 다음 토큰을 샘플링해 자기 입력에 붙이는 일을 반복하는 기계." 학생이 한 문장으로 이 흐름을 정리할 수 있게 하는 것이 6부 후반의 종착점.

STEP 1. 시퀀스 (Phase 19)

1-1순서가 의미를 만든다 — 정답
정답모든 행에서 "의미가 같지 않다". 같은 단어 집합이라도 누가 행위자(주어), 누가 대상(목적어)인지가 순서로 결정되기 때문.
1-2평균 임베딩 — 계산 정답
계산 "I love you" 평균 = ((0.9 + 0.0 + (−0.9))/3, (0.2 + 0.9 + 0.2)/3) = (0.0, 0.433)
"You love I" 평균 = ((−0.9 + 0.0 + 0.9)/3, (0.2 + 0.9 + 0.2)/3) = (0.0, 0.433)
→ 같다.
결론평균 임베딩은 순서 정보를 잃는다. 그래서 의미가 정반대인 두 문장도 같은 점으로 매핑된다. 이 한계가 RNN과 어텐션의 발상으로 이어진다.
1-3RNN 상태 갱신 — 정답
"I love you" (단어 = 0.9, 0.0, −0.9) h₁ = 0.5·0 + 0.5·0.9 = 0.45
h₂ = 0.5·0.45 + 0.5·0.0 = 0.225
h₃ = 0.5·0.225 + 0.5·(−0.9) = −0.3375
"You love I" (단어 = −0.9, 0.0, 0.9) h₁ = 0.5·0 + 0.5·(−0.9) = −0.45
h₂ = 0.5·(−0.45) + 0.5·0.0 = −0.225
h₃ = 0.5·(−0.225) + 0.5·0.9 = +0.3375
결론두 마지막 상태가 부호까지 정반대(−0.3375 vs +0.3375) → RNN은 순서 정보를 보존한다. (같은 단어 집합·다른 순서가 다른 마지막 상태로 이어짐.)
수업 팁RNN의 장점만 강조하지 말고, "긴 문장에서 앞 정보가 희미해지는 한계(첫 단어 0.9가 마지막 상태에서 0.1125 정도로 옅어짐)"를 표 위에서 함께 짚어줄 것 — 어텐션의 등장 동기로 자연스럽게 연결됨.

STEP 2. 어텐션 (Phase 20)

2-1어텐션 점수 — 계산 정답
정답 S("봤다", "고양이는") = 0.09 + 0.14 + 0.06 + 0.09 = 0.38
S("봤다", "쥐를") = 0.02 + 0.12 + 0.24 + 0.09 = 0.47
S("봤다", "봤다") = 0.01 + 0.04 + 0.09 + 0.81 = 0.95
실제 softmax(스케일 √d 생략, 단순화) softmax(0.38, 0.47, 0.95) ≈ (0.26, 0.28, 0.46). "봤다"가 자기 자신에 가장 큰 가중치를 줌 — self-attention의 흔한 현상이라 짚어줄 것.
수업 팁학생용 활동지에는 단순화된 (0.30, 0.30, 0.40)로 진행하도록 제시해 가중합 계산을 깔끔하게 만들었음. 시간 여유가 있는 반은 실제 softmax 값으로 다시 계산해보게 하면 좋다.
2-2가중합 — 계산 정답 (가중치 0.30, 0.30, 0.40 사용 시)
계산 O = 0.30·(0.9, 0.7, 0.2, 0.1) + 0.30·(0.2, 0.6, 0.8, 0.1) + 0.40·(0.1, 0.2, 0.3, 0.9)
= (0.27, 0.21, 0.06, 0.03) + (0.06, 0.18, 0.24, 0.03) + (0.04, 0.08, 0.12, 0.36)
= (0.37, 0.47, 0.42, 0.42)
변화 방향"봤다"의 원래 벡터 (0.1, 0.2, 0.3, 0.9)와 비교하면 4번째 차원이 0.9 → 0.42로 줄고, 1·2번째 차원이 커졌다. → "봤다"가 "고양이는"·"쥐를"의 의미를 흡수해 더 풍부한 표현이 됨.
자주 틀림가중합 계산에서 차원별로 따로 더해야 하는데 한 차원에 모두 더하는 실수. 표를 4열로 그려 차원별로 적게 시범할 것.
2-3"봤다" 행이 의미하는 것 — 모범 답안
예시"봤다"는 동사라 주체와 대상이 모두 필요하다. 그래서 두 명사("고양이는"·"쥐를")에 비슷한 비중으로 주목해야 의미가 완성된다. 어텐션은 문법적 역할을 학습된 가중치로 자동 잡아낸다.

STEP 3. 멀티헤드와 블록 (Phase 21)

3-14개 헤드 — 정답
헤드"줬다"가 가장 주목하는 단어
주격 헤드철수가
대상 헤드영희에게
목적어 헤드책을
전체 문장 헤드모두 비슷 (0.25 부근)
메시지한 헤드 = 한 가지 관계. 멀티헤드는 관계의 종류만큼 채널을 늘리는 일이다.
3-2블록 부품 — 정답
① 멀티헤드 어텐션
② 잔차 연결 (residual connection)
③ FFN (feed-forward network, 작은 MLP)
④ LayerNorm (층 정규화)
수업 팁잔차 연결의 직관: "원본 입력을 유지한 채 변화를 더한다" → 깊게 쌓아도 학습이 안 부서지는 이유. 학생이 외울 필요는 없고 "이런 게 있구나" 정도로 충분.
3-31층 vs 6층 — 모범 답안
예시 관찰 1층: 인접 단어 사이 직접 관계만 잡힘 (주어-동사 등).
3층: 두 단계 떨어진 관계도 잡힘. 문맥의 일부 추상화.
6층: 문장 전체의 의미·역할 구조가 하나의 표현으로 통합됨.
메시지"깊이 = 문맥을 모으는 단계 수." 깊을수록 더 멀리 있는 단어 관계와 더 추상적인 패턴을 다룰 수 있다.

STEP 4. GPT 샘플링 (Phase 22)

4-1"고양이는" 다음 — 전형 결과
예시상위 5개: 귀엽다(≈0.27), 잔다(≈0.16), 쥐를(≈0.05), 동물(≈0.04), 집사(≈0.03). 정확한 값은 앱 logit에 따라 다르지만, "귀엽다 / 잔다"가 압도적 1·2위인 분포가 자연스럽다.
4-2온도 — 정답
T1순위 확률 경향분포 모양
0.20.7~0.9매우 뾰족 — 1위에 거의 몰림
1.00.2~0.3중간 — 위 결과처럼 분산
2.00.05~0.10평탄 — 거의 균등
정답T가 작을수록 보수적(반복·뻔함). T가 클수록 의외성↑(엉뚱·환각 위험).
4-3·4-4같은 시작·다른 결과 / 창의성 — 모범 답안
왜 다른가매 단계 분포에서 샘플링(확률적으로 한 토큰 선택)을 하기 때문. 같은 분포라도 매번 다른 토큰이 뽑힐 수 있다.
"창의성"의 정체"학습된 분포에서 매번 다른 점을 뽑아내는 무작위성과, 그 분포의 모양을 조절하는 온도(temperature)의 합. 새로운 의미를 만들어내는 게 아니라 분포의 다양한 영역을 표본으로 가져오는 일이다."
수업 팁"창의성 = 무작위성"으로 단순화하지 않도록 주의. 학습된 분포의 품질이 90%, 샘플링 전략이 10%라는 비유가 안전.

STEP 5. 마무리 — 정답

정리흐름 빈칸 — 정답
토큰 → (임베딩) 벡터 → (멀티헤드) 어텐션 + FFN을 가진 트랜스포머 블록을 N층 통과
→ 마지막에 어휘에 대한 (확률 분포(logit·softmax))이 만들어지고, (샘플링)으로 다음 토큰을 뽑는다.
이 과정을 반복(자기회귀)해 문장 전체가 만들어진다.

관찰·수행 평가 루브릭

수준시퀀스 (P19)어텐션 (P20)트랜스포머 (P21)GPT 샘플링 (P22)
A 도달 평균 임베딩의 한계를 직접 계산으로 보임. RNN 상태 누적의 의미를 자기 말로 설명. 4차원 어텐션 점수와 가중합을 손으로 정확히 계산. 결과 벡터가 어떤 단어 쪽으로 가까워졌는지 해석. 4개 헤드의 의미 차이를 식별. 블록 부품 4개 모두 이름으로 답. 깊이 효과를 추상화 수준으로 설명. 온도와 분포 모양의 관계를 정확히 설명. "창의성"을 분포·샘플링으로 재정의.
B 진행 평균 임베딩 계산은 하지만 결론을 못 일반화. 점수는 계산하나 가중합 계산에서 차원 합산 실수. 헤드 차이는 알지만 블록 부품 일부 누락. 온도 효과는 알지만 자기회귀 개념을 못 짚음.
C 보충 순서가 의미를 만든다는 메시지를 흡수 못함. 내적·가중합 자체를 어려워함. 어텐션 한 가지만 알고 멀티헤드의 의의를 못 잡음. "창의성 = 무작위"로 단순화에 머무름.
전이 포인트여기까지가 22단계 전체의 마지막 코스다. 7번째 워크시트(종합 평가지)에서는 부별로 1~2문항씩 묶어 학생이 전체 흐름을 자기 안에서 다시 짜 맞출 수 있는지 점검한다.
